Aggregations & GROUP BY

Chapter 9 · Aggregations & GROUP BY

From Thousands of Rows
to Meaningful Summaries

Aggregation functions collapse many rows into a single answer. GROUP BY lets you do that per group. Together, they power every dashboard, report, and business insight you'll ever build.

What is Aggregation?

Instead of looking at every row, aggregation summarises data. Instead of seeing 50,000 orders, you see "Total orders in Mumbai: 12,400". That's aggregation.

🔢 COUNT(*) vs COUNT(column) — Key Difference
⚠️
This is a very common interview question!
COUNT(*) counts ALL rows including NULLs.
COUNT(column) counts only rows where that column is NOT NULL.
SQL
-- How many students enrolled in GUVI?
SELECT COUNT(*)           AS total_rows,
       COUNT(score)        AS students_who_attempted,
       COUNT(DISTINCT city) AS unique_cities
FROM   guvi_learners;
➕ SUM() — Total Revenue
SQL
-- Total course fee collected by GUVI this month
SELECT SUM(fee_paid)         AS total_revenue,
       SUM(fee_paid * 0.18) AS gst_collected
FROM   payments
WHERE  payment_date BETWEEN '2024-06-01' AND '2024-06-30';
📐 AVG() — Average Score by Batch
SQL
-- Average quiz score per batch (ignores NULL automatically)
SELECT batch_id,
       ROUND(AVG(quiz_score), 2) AS avg_score
FROM   guvi_learners
GROUP BY batch_id
ORDER BY avg_score DESC;
⬇️⬆️ MIN() and MAX()
SQL
-- Best and worst scores per course on Zomato platform
SELECT course_name,
       MIN(rating)  AS lowest_rating,
       MAX(rating)  AS highest_rating,
       MAX(rating) - MIN(rating) AS range_val
FROM   course_reviews
GROUP BY course_name;
🔗 GROUP_CONCAT() / STRING_AGG() — Join Values

Concatenates all values in a group into one string — great for showing a list per category.

MySQL — GROUP_CONCAT
-- List all students per batch as a comma-separated string
SELECT batch_id,
       GROUP_CONCAT(student_name ORDER BY student_name SEPARATOR ', ')
         AS student_list,
       COUNT(*) AS total_students
FROM   guvi_learners
GROUP BY batch_id;
PostgreSQL — STRING_AGG
-- Same in PostgreSQL
SELECT batch_id,
       STRING_AGG(student_name, ', ' ORDER BY student_name) AS student_list
FROM   guvi_learners
GROUP BY batch_id;

∅ NULL Handling in Aggregations

NULLs behave in a specific (and sometimes surprising) way in aggregate functions. Getting this wrong leads to incorrect reports.

🚨
Golden Rule of NULLs in AggregationsAll aggregate functions (SUM, AVG, MIN, MAX, COUNT(col)) skip NULL values. Only COUNT(*) includes them.
📊 Sample Data — GUVI Learners
student_idnamequiz_scorecity
1Priya85Mumbai
2RaviNULLDelhi
3Sneha92Chennai
4ArjunNULLMumbai

🧮 What Each Function Returns with NULLs
FunctionResultWhy
COUNT(*)5Counts all rows
COUNT(quiz_score)3Only rows where quiz_score is not NULL
SUM(quiz_score)25585 + 92 + 78 (NULLs skipped)
AVG(quiz_score)85.0255 ÷ 3 (NOT 255 ÷ 5 !)
MIN(quiz_score)78Smallest non-NULL
MAX(quiz_score)92Largest non-NULL
⚠️
AVG Trap!AVG(quiz_score) = 85, NOT 51 (255÷5). This means "students who did not attempt" are excluded from the average — which may or may not be what you want. Always clarify the business requirement.
🛠️ Handling NULLs Explicitly with COALESCE
SQL
-- Treat NULL scores as 0 in the average
SELECT AVG(COALESCE(quiz_score, 0)) AS avg_including_absent
FROM   guvi_learners;
-- Result: (85+0+92+0+78)/5 = 51.0

-- Count students who did NOT attempt
SELECT COUNT(*) - COUNT(quiz_score) AS absent_students
FROM   guvi_learners;

📦 GROUP BY — Single & Multiple Columns

GROUP BY collapses all rows with the same value in a column into a single group — then you apply an aggregate function to each group.

📊 After GROUP BY city
📖 GROUP BY Syntax
SQL
SELECT   group_column, AGG_FUNCTION(value_column)
FROM     table_name
WHERE    condition          -- filter rows BEFORE grouping
GROUP BY group_column
ORDER BY AGG_FUNCTION(value_column) DESC;
🏙️ GROUP BY Single Column — Sales by City (Zomato)
SQL
-- Total orders and revenue per city
SELECT   city,
         COUNT(*)       AS total_orders,
         SUM(amount)    AS total_revenue,
         AVG(amount)    AS avg_order_value
FROM     zomato_orders
WHERE    order_date >= '2024-01-01'
GROUP BY city
ORDER BY total_revenue DESC;
📅 GROUP BY Multiple Columns — Monthly Sales by City

You can group by more than one column — each unique combination becomes its own group.

SQL
-- Monthly sales by city — GROUP BY two columns
SELECT   city,
         MONTH(order_date)  AS month,
         COUNT(*)           AS orders,
         SUM(amount)        AS revenue
FROM     zomato_orders
GROUP BY city, MONTH(order_date)
ORDER BY city, month;

⚠️
Common Mistake!Every column in your SELECT that is NOT inside an aggregate function MUST appear in the GROUP BY clause. If you write SELECT city, student_name, COUNT(*) with GROUP BY city — that's an error! student_name must either be in GROUP BY or wrapped in an aggregate.

🔍 HAVING — Filter on Aggregated Results

WHERE filters individual rows before grouping. HAVING filters groups after aggregation. This is one of the most important distinctions in SQL.

🔵 WHERE — filters ROWS (before grouping)

Applied to individual rows. Cannot use aggregate functions here.

SELECT   city, COUNT(*)
FROM     orders
WHERE    amount > 500
-- ↑ filter rows first
GROUP BY city;
🟣 HAVING — filters GROUPS (after grouping)

Applied to groups. Can use aggregate functions here.

SELECT   city, COUNT(*)
FROM     orders
GROUP BY city
HAVING   COUNT(*) > 1000
-- ↑ filter groups after
💡
Memory trick 🧠"WHERE filters what goes IN. HAVING filters what comes OUT."
WHERE → rows going into the group. HAVING → groups coming out of GROUP BY.

🏢 GUVI Example — Batches with Average Score above 75
SQL
-- Find batches where average quiz score exceeds 75
SELECT   batch_id,
         COUNT(*)                      AS learners,
         ROUND(AVG(quiz_score), 2)    AS avg_score
FROM     guvi_learners
WHERE    quiz_score IS NOT NULL      -- remove absent students first
GROUP BY batch_id
HAVING   AVG(quiz_score) > 75      -- filter groups by their avg
ORDER BY avg_score DESC;
🚂 IRCTC Example — Popular Routes
SQL
-- Routes with more than 500 bookings and avg ticket > ₹800
SELECT   source_city,
         destination_city,
         COUNT(*)         AS total_bookings,
         AVG(ticket_price) AS avg_price
FROM     irctc_bookings
WHERE    booking_date >= '2024-01-01'
GROUP BY source_city, destination_city
HAVING   COUNT(*) > 500
   AND   AVG(ticket_price) > 800
ORDER BY total_bookings DESC;

↕️ ORDER BY with Aggregations

You can sort by aggregate function results directly — extremely useful for finding top-performers, highest sales, worst-rated, etc.

📊 Sorting by Aggregate Functions
SQL
-- Top 5 cities by total Zomato revenue
SELECT   city,
         COUNT(*)    AS orders,
         SUM(amount) AS revenue
FROM     zomato_orders
GROUP BY city
ORDER BY SUM(amount) DESC
LIMIT    5;
SQL
-- You can also ORDER BY alias (works in MySQL/PostgreSQL)
SELECT   city,
         SUM(amount) AS revenue
FROM     zomato_orders
GROUP BY city
ORDER BY revenue DESC;  -- alias works here!
🏫 GUVI — Learner Completion Rate (Sorted)
SQL
-- Completion rate per batch, best first
SELECT
  batch_id,
  COUNT(*)                                     AS total_learners,
  SUM(CASE WHEN completed = 'Y' THEN 1 ELSE 0 END) AS completed_count,
  ROUND(
    100.0 * SUM(CASE WHEN completed = 'Y' THEN 1 ELSE 0 END) / COUNT(*),
    1
  )                                              AS completion_pct
FROM     guvi_learners
GROUP BY batch_id
ORDER BY completion_pct DESC;
✅
CASE WHEN inside SUM — a powerful pattern!SUM(CASE WHEN condition THEN 1 ELSE 0 END) counts rows matching a condition. This is how you calculate counts for specific values without a subquery.

📊 ROLLUP & GROUPING SETS

ROLLUP automatically adds subtotals and a grand total row to your GROUP BY results — perfect for financial and analytics reports.

💡
What is ROLLUP?Imagine GROUP BY gives you sales per city per month. ROLLUP additionally gives you:
→ Total sales per city (across all months)
→ Grand total (across all cities and months)
📋 ROLLUP Syntax
MySQL / PostgreSQL
SELECT   city, MONTH(order_date) AS month, SUM(amount) AS revenue
FROM     zomato_orders
GROUP BY city, MONTH(order_date) WITH ROLLUP;
-- MySQL syntax: WITH ROLLUP at end of GROUP BY

-- PostgreSQL syntax:
GROUP BY ROLLUP(city, MONTH(order_date));

🔍 Use GROUPING() to Label NULL vs Actual NULL
SQL
-- GROUPING() returns 1 if NULL came from ROLLUP, 0 if real NULL
SELECT
  CASE WHEN GROUPING(city) = 1 THEN 'ALL CITIES' ELSE city END AS city,
  CASE WHEN GROUPING(month) = 1 THEN 'ALL MONTHS' ELSE CAST(month AS CHAR) END AS month,
  SUM(amount) AS revenue
FROM     zomato_orders
GROUP BY city, month WITH ROLLUP;
🧩 GROUPING SETS — Custom Grouping Combinations

GROUPING SETS lets you define exactly which combinations you want — more flexible than ROLLUP.

PostgreSQL / SQL Standard
-- Get totals by (city, month), by city alone, and grand total
SELECT   city, month, SUM(amount)
FROM     zomato_orders
GROUP BY GROUPING SETS (
  (city, month),  -- group by both
  (city),         -- group by city only
  ()              -- grand total
);
🎯
When to use what?
ROLLUP → hierarchical data (year → month → day, country → city → branch)
GROUPING SETS → custom combinations (want city total but NOT month total)

🇮🇳 Real-World Examples

Three complete case studies: monthly sales by city, average score by batch, and learner completion rates — all using Indian datasets.

🍕 Case 1 — Monthly Sales by City (Zomato)
SQL
-- Monthly order summary by city
SELECT
  city,
  MONTHNAME(order_date)       AS month_name,
  COUNT(*)                   AS total_orders,
  SUM(amount)               AS total_revenue,
  ROUND(AVG(amount), 2)     AS avg_order_value,
  MAX(amount)               AS largest_order
FROM   zomato_orders
WHERE  YEAR(order_date) = 2024
GROUP BY city, MONTHNAME(order_date), MONTH(order_date)
HAVING   COUNT(*) > 100
ORDER BY city, MONTH(order_date);

📚 Case 2 — Average Score by Batch (GUVI)
SQL
-- Performance report per batch — only batches with 10+ learners
SELECT
  batch_id,
  COUNT(*)                    AS total_learners,
  COUNT(quiz_score)           AS attempted,
  ROUND(AVG(quiz_score), 1)  AS avg_score,
  MIN(quiz_score)             AS lowest,
  MAX(quiz_score)             AS highest,
  GROUP_CONCAT(
    CASE WHEN quiz_score >= 90
    THEN student_name END
    ORDER BY quiz_score DESC
    SEPARATOR ', '
  )                           AS top_scorers
FROM   guvi_learners
GROUP BY batch_id
HAVING   COUNT(*) >= 10
ORDER BY avg_score DESC;
📈 Case 3 — Learner Completion Rates (GUVI)
SQL
-- Course completion rates — with HAVING to flag struggling batches
SELECT
  course_name,
  batch_id,
  COUNT(*)                                        AS enrolled,
  SUM(CASE WHEN status = 'Completed' THEN 1 ELSE 0 END) AS completed,
  SUM(CASE WHEN status = 'Dropped'   THEN 1 ELSE 0 END) AS dropped,
  ROUND(
    100.0 * SUM(CASE WHEN status = 'Completed' THEN 1 ELSE 0 END)
    / COUNT(*), 1
  )                                                AS completion_pct
FROM   guvi_enrollments
GROUP BY course_name, batch_id
HAVING   completion_pct < 60       -- flag batches needing attention
ORDER BY completion_pct ASC;       -- worst first
