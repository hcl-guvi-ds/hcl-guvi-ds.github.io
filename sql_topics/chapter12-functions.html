String, Date & Math Functions

Chapter 12 · Built-in Functions

SQL's Toolbox.
Transform, Format & Calculate.

Raw data is messy — names have inconsistent casing, dates are stored as strings, phone numbers have dashes. Built-in functions let you clean, transform, and compute directly inside your SQL query, no Python needed.

What are Built-in Functions?

SQL comes with hundreds of pre-built functions that operate on data at the row level. Unlike aggregate functions (COUNT, SUM), these work on individual column values — transforming or extracting information from each row.

🔤 String Functions

Clean, split, pad, search, and format text. Essential for names, emails, phone numbers, addresses.

📅 Date Functions

Parse dates, calculate age, find days between events, extract month/year/quarter.

🔢 Math Functions

Round, truncate, compute modulo, handle NULL values in arithmetic calculations.

🎓
Real-world contextAt GUVI, student names come in as "priya SHARMA", " RAVI kumar " — inconsistent casing, extra spaces. Date of birth is stored as '15-Aug-2000'. Phone numbers look like '91-9876543210'. Built-in functions fix all of this directly in SQL.

🏫 GUVI — Cleaning Student Names
SQL
-- Raw data: '  priya SHARMA  ' → cleaned: 'Priya Sharma'
SELECT
  name                                    AS raw_name,
  TRIM(name)                              AS trimmed,
  UPPER(TRIM(name))                       AS uppercased,
  CONCAT(
    UPPER(LEFT(TRIM(name),1)),
    LOWER(SUBSTRING(TRIM(name),2))
  )                                       AS title_case
FROM guvi_students;
🔗 CONCAT_WS — Build Formatted Strings
SQL
-- Build full address from components
SELECT
  CONCAT_WS(', ', street, city, state, pincode) AS full_address
FROM   students;
-- Output: '14 MG Road, Chennai, TN, 600001'

-- CONCAT_WS skips NULLs automatically! Safe for optional fields
SELECT CONCAT_WS(' ', first_name, middle_name, last_name) AS full_name
FROM   students;
-- 'Priya Sharma' (middle_name NULL → skipped, no double space)
💡
CONCAT_WS vs CONCATCONCAT_WS(sep, a, b, c) automatically skips NULL values. CONCAT(a, b, c) returns NULL if any argument is NULL — a nasty surprise in production!
✂️ SUBSTRING — Extracting Parts of Strings
SQL
-- SUBSTRING(string, start, length)  ← 1-indexed!
SELECT
  batch_name,
  SUBSTRING(batch_name, 1, 4)  AS course_code,  -- 'AIML'
  RIGHT(batch_name, 4)          AS batch_year,   -- '2024'
  LENGTH(batch_name)             AS name_length
FROM   batches;
-- 'AIML-Jun-2024' → course_code: AIML | year: 2024 | len: 13

📞 REPLACE — Cleaning Phone Numbers
SQL
-- Remove dashes and spaces from phone numbers
SELECT
  phone                                       AS raw_phone,
  REPLACE(REPLACE(phone, '-', ''), ' ', '') AS clean_phone
FROM   students;
-- '91-98765-43210'  →  '919876543210'

-- Standardise email domain
SELECT REPLACE(email, '@gmail.com', '@guvi.in') AS guvi_email
FROM   students;
📧 INSTR — Extract Username from Email
SQL
-- Get username part of email (everything before @)
SELECT
  email,
  LEFT(email, INSTR(email, '@') - 1) AS username
FROM   students;
-- 'priya@guvi.in'  →  username: 'priya'
-- INSTR returns position 6 of '@', LEFT takes chars 1-5

🧩 REGEXP / RLIKE — Pattern Matching

REGEXP (or RLIKE in MySQL) lets you match strings against regular expression patterns. Far more powerful than LIKE for complex validation.

🎯
LIKE vs REGEXP
LIKE 'priya%' → simple wildcard matching
REGEXP '^[A-Z]' → full regex power — anchors, character classes, quantifiers

🏫 GUVI — Validate Emails & Phone Numbers
SQL
-- Find students with invalid email format (missing @ or .)
SELECT name, email
FROM   students
WHERE  email NOT REGEXP '^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$';

-- Find students whose names contain only letters and spaces
SELECT name
FROM   students
WHERE  name REGEXP '^[A-Za-z ]+$';

-- Find phone numbers that are NOT 10 digits
SELECT name, phone
FROM   students
WHERE  phone NOT REGEXP '^[0-9]{10}$';

🎂 Calculate Age from Date of Birth
SQL
-- Method 1: Using TIMESTAMPDIFF (most accurate)
SELECT
  name,
  dob,
  TIMESTAMPDIFF(YEAR, dob, CURDATE()) AS age_years
FROM   guvi_students;
-- dob = '2000-08-15' → age_years = 23 (as of 2024-06-15)

-- Method 2: Using DATEDIFF and FLOOR
SELECT
  name,
  FLOOR(DATEDIFF(CURDATE(), dob) / 365.25) AS approx_age
FROM   guvi_students;
📋 DATE_FORMAT — Display Dates Beautifully
SQL
-- Format dates for Indian display style
SELECT
  enrollment_date,
  DATE_FORMAT(enrollment_date, '%d/%m/%Y')      AS indian_fmt,   -- 15/06/2024
  DATE_FORMAT(enrollment_date, '%d %M %Y')      AS long_fmt,     -- 15 June 2024
  DATE_FORMAT(enrollment_date, '%W, %d %b %Y')  AS full_fmt      -- Saturday, 15 Jun 2024
FROM   enrollments;

-- Parse a string date into proper DATE type
SELECT STR_TO_DATE('15-Aug-2000', '%d-%b-%Y') AS proper_date;
-- Output: 2000-08-15

⏱️ Date Arithmetic & Fiscal Quarters

Calculate days between dates, add/subtract time intervals, find days since last login, and label fiscal quarters.

⏳ DATEDIFF, DATE_ADD, DATE_SUB
SQL
-- Days since last login (engagement metric)
SELECT
  name,
  last_login,
  DATEDIFF(CURDATE(), last_login)  AS days_since_login,
  CASE
    WHEN DATEDIFF(CURDATE(), last_login) <= 7  THEN '🟢 Active'
    WHEN DATEDIFF(CURDATE(), last_login) <= 30 THEN '🟡 Idle'

FROM   guvi_students;

-- DATE_ADD / DATE_SUB
SELECT
  enrollment_date,
  DATE_ADD(enrollment_date, INTERVAL 90 DAY)   AS trial_ends,
  DATE_SUB(enrollment_date, INTERVAL 1 MONTH)  AS one_month_before,
  DATE_ADD(enrollment_date, INTERVAL 1 YEAR)   AS renewal_date
FROM   subscriptions;

🏦 Fiscal Quarter Labelling — Indian Financial Year

India's financial year runs April to March. Q1 = Apr–Jun, Q2 = Jul–Sep, Q3 = Oct–Dec, Q4 = Jan–Mar. This is different from the calendar year quarter.

SQL — Indian Fiscal Quarter
-- Indian FY: Q1=Apr-Jun, Q2=Jul-Sep, Q3=Oct-Dec, Q4=Jan-Mar
SELECT
  payment_date,
  YEAR(payment_date)  AS cal_year,
  MONTH(payment_date) AS month,
  CASE
    WHEN MONTH(payment_date) IN (4,5,6)    THEN 'Q1 (Apr-Jun)'
    WHEN MONTH(payment_date) IN (7,8,9)    THEN 'Q2 (Jul-Sep)'
    WHEN MONTH(payment_date) IN (10,11,12) THEN 'Q3 (Oct-Dec)'
    ELSE                                       'Q4 (Jan-Mar)'
  END                 AS fiscal_quarter,
  CASE
    WHEN MONTH(payment_date) >= 4
    THEN CONCAT('FY', YEAR(payment_date), '-', YEAR(payment_date)+1)
    ELSE CONCAT('FY', YEAR(payment_date)-1, '-', YEAR(payment_date))
  END                 AS fiscal_year
FROM   payments
ORDER BY payment_date;

💰 Finance at HDFC — Rounding & Calculations
SQL
-- Calculate EMI and round correctly for bank display
SELECT
  loan_amount,
  interest_rate,
  ROUND(loan_amount * interest_rate / 100, 2)  AS interest_rs,
  CEIL(loan_amount / 12)                       AS monthly_emi_ceil,
  FLOOR(loan_amount / 12)                      AS monthly_emi_floor
FROM   hdfc_loans;

-- MOD: Identify even/odd student IDs for A/B testing
SELECT
  student_id,
  name,
  CASE WHEN MOD(student_id, 2) = 0
       THEN 'Group A' ELSE 'Group B'
  END AS ab_group
FROM   guvi_students;

-- ROUND vs TRUNCATE — important difference!
-- ROUND(4.987, 2) = 4.99  (rounds up because .7 ≥ 5)
-- TRUNCATE(4.987, 2) = 4.98  (just cuts off, no rounding)
💡
CEIL vs FLOOR — When to use which?
CEIL → "round up to be safe" — e.g., pages needed, seats to book, number of buses required
FLOOR → "round down conservatively" — e.g., age in years, completed months, installments paid

∅ COALESCE & NULLIF — Elegant NULL Handling

Two of the most useful functions in SQL for handling NULL values gracefully in production code.

COALESCE(a, b, c, ...)

Returns the first non-NULL value from the list. Provide a fallback for missing data.

NULLIF(a, b)

Returns NULL if a = b, otherwise returns a. Prevents division-by-zero by converting 0 to NULL.

COALESCE — Fallback Chain
SQL
-- Show best available contact: mobile first, then email, then 'No Contact'
SELECT
  name,
  COALESCE(mobile, email, whatsapp, 'No Contact Available') AS best_contact
FROM   guvi_students;

-- Replace NULL scores with 0 for display
SELECT
  name,
  COALESCE(quiz_score, 0) AS score_display,
  COALESCE(city, 'Unknown City') AS city
FROM   guvi_students;

-- Show batch name or 'Unassigned' if no batch yet
SELECT
  s.name,
  COALESCE(b.batch_name, 'Not Yet Assigned') AS batch
FROM   students s
LEFT JOIN batches b ON s.batch_id = b.batch_id;
NULLIF — The Division-by-Zero Saviour
SQL
-- Without NULLIF: division by zero error if attempts = 0!
SELECT correct_answers / total_attempts AS accuracy  -- ❌ crashes if attempts=0

-- With NULLIF: returns NULL instead of crashing
SELECT
  name,
  correct_answers,
  total_attempts,
  ROUND(
    correct_answers / NULLIF(total_attempts, 0) * 100,
    1
  ) AS accuracy_pct    -- returns NULL if attempts=0 (safe!)
FROM   quiz_results;

-- NULLIF for blank string handling
SELECT COALESCE(NULLIF(TRIM(city), ''), 'Unknown') AS city
FROM   students;
-- TRIM removes spaces, NULLIF converts '' to NULL, COALESCE fills 'Unknown'
✅
Power combo: COALESCE(NULLIF(TRIM(col), ''), 'default')
This 3-function chain handles three data quality issues in one line: leading/trailing spaces (TRIM), empty string after trim (NULLIF → NULL), and NULL values (COALESCE → default).

🇮🇳 Real-World Recipes

Four complete, production-ready queries combining multiple functions to solve real business problems at GUVI, IRCTC, Zomato, and HDFC.

📊 Recipe 1 — Complete Student Profile Report (GUVI)
SQL — Combines: CONCAT_WS, TRIM, UPPER, DATE_FORMAT, DATEDIFF, COALESCE, TIMESTAMPDIFF
SELECT
  -- Clean full name
  CONCAT_WS(' ',
    UPPER(LEFT(TRIM(first_name),1)) || LOWER(SUBSTRING(TRIM(first_name),2)),
    UPPER(LEFT(TRIM(last_name),1))  || LOWER(SUBSTRING(TRIM(last_name),2))
  )                                      AS full_name,

  -- Age
  TIMESTAMPDIFF(YEAR, dob, CURDATE())  AS age,

  -- Enrollment date formatted
  DATE_FORMAT(enrolled_on, '%d %M %Y')  AS enrolled_on_fmt,

  -- Days since joined
  DATEDIFF(CURDATE(), enrolled_on)       AS days_on_platform,

  -- Last login status
  COALESCE(
    CONCAT(DATEDIFF(CURDATE(), last_login), ' days ago'),
    'Never logged in'
  )                                      AS last_seen,

  -- Clean phone
  REPLACE(REPLACE(phone, '-', ''), ' ', '') AS clean_phone,

  -- Score with fallback
  COALESCE(ROUND(quiz_score,1), 0)      AS score

FROM   guvi_students
ORDER BY enrolled_on DESC;
🚂 Recipe 2 — IRCTC: Days Until Journey & PNR Format
SQL
SELECT
  LPAD(pnr_number, 10, '0')                         AS formatted_pnr,
  UPPER(passenger_name)                             AS passenger,
  DATE_FORMAT(journey_date, '%W, %d %b %Y')        AS journey_day,
  DATEDIFF(journey_date, CURDATE())                AS days_until_journey,
  CONCAT(source_city, ' → ', destination_city)      AS route,
  FORMAT(ticket_price, 0)                           AS price_inr,
  COALESCE(seat_number, 'Waitlisted')              AS seat
FROM   irctc_bookings
WHERE  journey_date >= CURDATE()
ORDER BY journey_date;
🍕 Recipe 3 — Zomato: Order Summary with Timestamps
SQL
SELECT
  order_id,
  restaurant_name,
  DATE_FORMAT(order_time, '%d/%m/%Y %H:%i')       AS ordered_at,
  DATE_FORMAT(delivered_time, '%H:%i')             AS delivered_at,
  TIMESTAMPDIFF(MINUTE, order_time, delivered_time) AS delivery_mins,
  FORMAT(total_amount, 2)                           AS total,
  CASE
    WHEN TIMESTAMPDIFF(MINUTE, order_time, delivered_time) <= 30
    THEN '🟢 On Time'
    WHEN TIMESTAMPDIFF(MINUTE, order_time, delivered_time) <= 45
    THEN '🟡 Slight Delay'
    ELSE '🔴 Late'
  END                                               AS delivery_status
FROM   zomato_orders
WHERE  DATE(order_time) = CURDATE();
🏦 Recipe 4 — HDFC: Loan Age & Safe Division
SQL
SELECT
  account_id,
  UPPER(customer_name)                                         AS customer,
  FORMAT(loan_amount, 0)                                      AS loan_amt,
  TIMESTAMPDIFF(MONTH, loan_start_date, CURDATE())           AS months_active,
  ROUND(
    COALESCE(amount_paid, 0) / NULLIF(loan_amount, 0) * 100,
    1
  )                                                            AS pct_repaid,
  DATE_FORMAT(
    DATE_ADD(loan_start_date, INTERVAL tenure_months MONTH),
    '%d %M %Y'
  )                                                            AS maturity_date
FROM   hdfc_loans
ORDER BY pct_repaid DESC;
