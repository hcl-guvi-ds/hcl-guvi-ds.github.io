Tokenisation — Splitting Text into Units

Before any NLP algorithm can work, raw text must be broken into pieces called tokens. Tokens can be words, subwords, characters, or sentences. This is always Step 1.

jigsaw

Analogy — Jigsaw puzzle pieces: Imagine a sentence as a completed puzzle. Tokenisation dismantles it into individual pieces (tokens). Word tokenisation makes each word a piece. Subword tokenisation (used by BERT) splits rare words like "unhappiness" into "un", "happiness" — so even unknown words can be processed using known subwords.

Click any button above to tokenise your text!
Types of Tokenisation
W
Word
Click
Split on spaces and punctuation. "Hello world!" becomes ["Hello","world","!"]. Simplest and most common. Problem: "run", "running", "ran" = 3 different tokens.
SW
Subword (BPE)
Click
Byte-Pair Encoding splits rare words into frequent subwords. "unhappiness" = ["un","##happiness"]. Used by BERT, GPT. Balances vocabulary size and coverage.
C
Character
Click
Each character is a token. "cat" = ["c","a","t"]. Very long sequences but zero unknown tokens. Used in character-level language models and some multilingual models.
S
Sentence
Click
Split paragraphs into sentences. Uses punctuation patterns (. ! ?). Needed before word tokenisation in many pipelines. NLTK's sent_tokenize() handles edge cases like "Dr. Smith".

python
import nltk
from nltk.tokenize import (
    word_tokenize, sent_tokenize,
    TreebankWordTokenizer,
    TweetTokenizer
)

# Word tokenisation

print(words)

# ["It's amazing!", "Dr. Smith won."]

# BERT subword tokenisation

print(tokens)  # ["un", "##happiness"]

Stop Words — Removing the Noise

Stop words are high-frequency words ("the", "is", "and", "a") that carry almost no meaningful information. Removing them reduces noise and lets algorithms focus on content words.

filter

Analogy — Photography noise reduction: A photo has content (what you want to see) and noise (random pixels that distract). Stop words are the noise — they appear in every document and carry no discriminating power. Removing them is like applying a noise filter: the meaningful content becomes clearer.

Common English Stop Words
Warning: sometimes stop words matter! "not good" vs "good" — removing "not" changes the meaning completely. In sentiment analysis, keep negations!

Stemming & Lemmatisation

Both techniques reduce words to their base form so "running", "runs", "ran" are all treated as the same concept. But they work very differently and produce different results.

scissors

Analogy — Two types of haircuts: Stemming is a rough chop — it mechanically strips suffixes. "Studies" becomes "studi" (not a real word!). Lemmatisation is a professional styling — it uses a dictionary to find the proper base form. "Studies" becomes "study", "better" becomes "good". Lemmatisation is slower but smarter.

for w in words:
    stem = porter.stem(w)

for token in doc:

# flying (verb) -> fly

POS Tagging — Parts of Speech

POS (Part-of-Speech) tagging assigns a grammatical role to each word: noun, verb, adjective, etc. This helps machines understand sentence structure and word meaning in context.

label

Analogy — Job roles in a company: In a company, everyone has a role: CEO (noun), manages (verb), efficiently (adverb). POS tagging gives each word its "job title" in the sentence. The word "bank" might be a noun (river bank) or verb (bank on it) — POS tagging uses context to decide.

Bag of Words — Counting Words

Bag of Words (BoW) converts text into numerical vectors by counting word occurrences. It ignores grammar and word order — treats text as an unordered collection of words.

bag

Analogy — Shaking a book into a bag: Imagine tearing out all the words from a book and shaking them into a bag. You lose the order and grammar, but you can count how often each word appears. "Machine learning is great and learning is fun" becomes {machine:1, learning:2, great:1, fun:1}. Simple but powerful for classification.

Topic 7 of 11

TF-IDF — Smarter Word Weighting

TF-IDF (Term Frequency - Inverse Document Frequency) fixes BoW's problem: common words like "the" get high counts but low importance. TF-IDF down-weights common words and up-weights rare, distinctive ones.

scale

Analogy — Hotel reviews: In 1000 hotel reviews, the word "hotel" appears in ALL of them — it tells you nothing. But "bedbug" appears in only 3 reviews — those reviews are clearly about something important! TF-IDF gives high weight to words that appear often in ONE document but rarely across ALL documents.

TF-IDF Calculator Interactive

Enter text documents and compute TF-IDF scores interactively:

The TF-IDF Formula
TF-IDF = TF(t,d) x IDF(t)
TF = count(t in d) / total words in d
IDF = log(N / df(t))
TF: how often term appears in THIS document. IDF: log of (total docs / docs containing term). High TF-IDF = word is frequent here but rare everywhere else.
Python Code Code
python
from sklearn.feature_extraction.text import TfidfVectorizer
import pandas as pd

docs = [
    "data science machine learning",
    "machine learning deep learning neural",
    "data analysis statistics science"
]

vec = TfidfVectorizer(smooth_idf=True)
X = vec.fit_transform(docs)
features = vec.get_feature_names_out()

df = pd.DataFrame(X.toarray().round(3),
    columns=features,
    index=[f"Doc{i+1}" for i in range(len(docs))])
print(df)

# Top TF-IDF term per document
for i in range(len(docs)):
    top = df.iloc[i].idxmax()
    print(f"Doc{i+1} key term: {top}")

>>>Doc2: 0.000 0.000 0.522 0.525 0.346 0.522 0.000 0.000
>>>Doc3: 0.522 0.346 0.000 0.000 0.000 0.000 0.346 0.522

Word Embeddings — Words as Vectors

Word embeddings map words to dense numeric vectors where similar words are close together in vector space. "king" - "man" + "woman" = "queen". Semantic meaning is captured mathematically.

compass

Analogy — City map coordinates: BoW gives words unique IDs (king=1, queen=2 — no relationship). Embeddings are like GPS coordinates: king=(3.2, -1.1, 0.8...), queen=(3.1, -0.9, -0.6...). They are geographically close! Words with similar meanings live near each other in 300-dimensional space.

Embedding Models Compared
Word2Vec (Google, 2013)
Predicts context from word (Skip-gram) or word from context (CBOW). 300-dimensional vectors. Fast training.
GloVe (Stanford, 2014)
Global co-occurrence matrix factorisation. Better at capturing global statistics. 50-300 dimensions.
FastText (Facebook, 2016)
Adds character n-grams. Handles misspellings and out-of-vocabulary words. Best for morphologically rich languages.
BERT Embeddings (2018+)
Contextual embeddings — "bank" has different vector in "river bank" vs "bank account". Dynamic, not static!

Topic 9 of 11

Sentiment Analysis — Mining Opinions

Sentiment Analysis automatically determines whether text expresses a positive, negative, or neutral opinion. It powers product reviews, social media monitoring, and customer feedback systems.

heart

Analogy — Restaurant critic vs survey: Reading 10,000 restaurant reviews manually = impossible. Sentiment analysis is like a tireless critic who reads every review and scores it: "The pasta was divine but the service was shockingly rude" = Mixed (positive food, negative service). Modern models go beyond positive/negative to detect anger, joy, fear, sarcasm.

Live Sentiment Analyser Interactive
Sentiment Approaches
1
Lexicon-Based (VADER)
Uses a hand-crafted dictionary of sentiment scores. "amazing"=+3.1, "terrible"=-2.5. Handles negation ("not good") and intensifiers ("very good"). Fast, no training data needed.
2
Machine Learning (TF-IDF + classifier)
Train a classifier (Naive Bayes, SVM, LogReg) on labelled review data. TF-IDF features + labels (positive/negative). Better accuracy than lexicon for domain-specific text.
3
Deep Learning (LSTM/BERT)
Fine-tune BERT on sentiment task. Captures context, sarcasm, and nuance better than bag-of-words. State-of-the-art accuracy. "Not bad" correctly maps to positive!
4
Aspect-Based Sentiment
Identifies sentiment for specific aspects. "The food was great but service was terrible" -> food=positive, service=negative. Used in product reviews and hotel feedback analysis.
Python Code Code
python
from nltk.sentiment import SentimentIntensityAnalyzer
from transformers import pipeline

# VADER (rule-based, great for social media)

scores = sia.polarity_scores(text)
print(scores)
# {'neg':0.0,'neu':0.252,'pos':0.748,'compound':0.8754}
# compound >= 0.05 = positive
# compound <= -0.05 = negative

# BERT-based (handles context and nuance)
sentiment = pipeline("sentiment-analysis")
result = sentiment("Not bad at all, I quite liked it")
print(result)
# [{'label':'POSITIVE','score':0.9987}]
# Correctly handles "Not bad" = positive!

# Aspect-based with opinion mining
absa = pipeline("text-classification",
    model="yangheng/deberta-v3-base-absa-v1.1")
r = absa("The food was great but slow service",
    text_pair="food")
print(f"Food sentiment: {r}")
>>>VADER: {'neg':0.0, 'neu':0.252, 'pos':0.748, 'compound':0.8754}
>>>BERT: [{'label':'POSITIVE', 'score':0.9987}]
>>>Aspect 'food': POSITIVE (0.9732)
>>>Aspect 'service': NEGATIVE (0.9541)
Aspect-based separates opinions for different product features!

Language Models — Predicting the Next Word

A language model assigns probabilities to sequences of words. Given "The cat sat on the ___", it predicts "mat" is more likely than "car". From n-grams to GPT, this is the core of generative AI.

crystal

Analogy — Autocomplete on steroids: Your phone's autocomplete suggests the next word based on the last few words you typed (n-gram). GPT does the same but considers your entire conversation history (transformer with full context window). Language models learned the statistics of language from billions of sentences.

Next-Word Predictor — Try It! Interactive
Evolution of Language Models
N-gram Models (before 2010)
Count n-word sequences. P(the|cat sat on) = count("cat sat on the") / count("cat sat on"). Simple, fast, but needs huge data and fails on long dependencies.
RNN / LSTM (2013-2017)
Recurrent neural networks process sequences step by step, maintaining hidden state. LSTMs solve the vanishing gradient problem. Can model longer contexts but still struggle with very long sequences.
Attention + Seq2Seq (2015-2018)
Attention mechanism lets the model focus on relevant parts of the input. "Attention is All You Need" (2017) introduced the Transformer architecture — the foundation of all modern LLMs.
GPT / BERT / LLMs (2018-now)
Transformer-based pre-trained models with billions of parameters. GPT-4, LLaMA, Gemini — trained on the entire internet. Fine-tune for any NLP task with minimal labelled data.

Attention & BERT Transformers

The Attention mechanism lets models focus on relevant words when processing each word. BERT uses bidirectional attention to understand context from both left and right simultaneously — revolutionising every NLP task.

spotlight

Analogy — Spotlight in a theatre: When an actor speaks a line, they don't pay equal attention to every other actor. The spotlight (attention) focuses on whoever is most relevant. In "The bank near the river flooded", to understand "bank" the model shines its spotlight on "river" (high attention weight), not "flooded". This dynamic focusing is the key innovation.

Complete Text Preprocessing Pipeline
Build a function that takes raw text and applies: lowercase, remove punctuation, tokenise, remove stop words, and lemmatise. Test on a product review. Report token count before and after each step.
python
import nltk, re, string
from nltk.tokenize import word_tokenize
from nltk.corpus import stopwords
from nltk.stem import WordNetLemmatizer

lemmatizer = WordNetLemmatizer()
stop_words = set(stopwords.words('english'))

def preprocess_pipeline(text):
    steps = {}
    steps['original'] = text
    text = text.lower()
    steps['lowercase'] = text
    text = re.sub(r'[^\w\s]', '', text)
    steps['no_punctuation'] = text
    tokens = word_tokenize(text)
    steps['tokenised'] = tokens
    tokens = [t for t in tokens if t not in stop_words]
    steps['no_stopwords'] = tokens
    tokens = [lemmatizer.lemmatize(t) for t in tokens]
    steps['lemmatised'] = tokens
    for step, result in steps.items():
        count = len(result) if isinstance(result, list) else len(result.split())
        print(f"{step:20s}: {count:3d} tokens -- {str(result)[:60]}")
    return tokens

review = "The product is absolutely amazing! I've been using it for years and it still works perfectly. Highly recommended to everyone."
clean_tokens = preprocess_pipeline(review)

Build a Text Classifier with TF-IDF + SVM

    train_dataset=tokenised['train'].select(range(2000)),
    eval_dataset=tokenised['test'].select(range(500)))
trainer.train()
results = trainer.evaluate()
