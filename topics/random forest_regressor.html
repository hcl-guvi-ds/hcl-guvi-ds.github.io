Why Random Forest?

Powerful, robust, and works out-of-the-box on many problems. Random Forest reduces overfitting through ensemble averaging and handles non-linear relationships naturally.

Like a jury making a verdict: One person might be biased, but multiple jury members vote and reach consensus. Ensemble methods work the same way—multiple models vote to make better decisions!

Bagging (Bootstrap Aggregating)
Creates multiple subsets through random sampling WITH replacement. Trains separate models on each subset and averages predictions. Reduces variance.

Python from sklearn.ensemble import BaggingRegressor from sklearn.tree import DecisionTreeRegressor # Create bagging ensemble base_tree = DecisionTreeRegressor(max_depth=5) bagging_model = BaggingRegressor( estimator=base_tree, n_estimators=10, bootstrap=True ) bagging_model.fit(X_train, y_train) predictions = bagging_model.predict(X_test)

Compare with Single Tree

Train both a Decision Tree and a Random Forest. Compare their R² scores.

from sklearn.tree import DecisionTreeRegressor

dt = DecisionTreeRegressor(max_depth=15)
dt.fit(X_train, y_train)
rf = RandomForestRegressor(n_estimators=100)
rf.fit(X_train, y_train)
print(f"Decision Tree R²: {dt.score(X_test, y_test):.4f}")
print(f"Random Forest R²: {rf.score(X_test, y_test):.4f}")
3
Hyperparameter Tuning

Use GridSearchCV to find optimal parameters for Random Forest.

from sklearn.model_selection import GridSearchCV

param_grid = {'n_estimators': [50, 100, 200], 'max_depth': [10, 15, 20]}
grid = GridSearchCV(RandomForestRegressor(), param_grid, cv=5)
grid.fit(X_train, y_train)
print(f"Best params: {grid.best_params_}")
