📐 Concept 1

What is Polynomial Regression?

When a straight line can't capture the pattern in your data, polynomial regression bends the line into a curve by adding higher-degree terms.

💡
Key idea: Polynomial regression is still linear regression at heart — it's linear in the coefficients. We just transform the features first: X → X, X², X³, …
📏 From Linear to Polynomial

Linear regression fits a straight line: ŷ = β₀ + β₁x. Polynomial regression fits a curve:

ŷ = β₀ + β₁x + β₂x² + β₃x³ + … + βₙxⁿ

The degree n controls how "curved" the fit is. Degree 1 = linear. Degree 2 = quadratic (parabola). Degree 3 = cubic, etc.

🐍 Python Example
import numpy as np from sklearn.preprocessing import PolynomialFeatures from sklearn.linear_model import LinearRegression # Sample data: house age vs price (non-linear relationship) X = np.array([[1],[2],[3],[4],[5],[6],[7]]) y = np.array([100,140,165,175,165,145,110]) # Step 1: Transform features (degree=2 → adds x²) poly = PolynomialFeatures(degree=2) X_poly = poly.fit_transform(X) # Step 2: Fit linear regression on transformed features model = LinearRegression() model.fit(X_poly, y) # Step 3: Predict pred = model.predict(poly.transform([[4]])) print(f"Predicted price at year 4: {pred[0]:.2f}") print(f"Coefficients: {model.coef_}")

🎛️ Concept 2

Degree & Fit Quality

The polynomial degree is the most important hyperparameter. Too low = underfit. Too high = overfit. Goldilocks = just right!

📉 Underfitting (Low Degree)

When degree is too low (e.g., 1 or 2), the model is too simple to capture the real pattern. High training AND test error — the model is biased.

⚠️
Signs: High MSE, low R², model ignores curvature in data.
📈 Overfitting (High Degree)

When degree is too high (e.g., 9+), the model memorizes training noise. Perfect on train data, terrible on test data — high variance.

⚠️
Signs: Near-zero train error, high test error, wild oscillations.
👗
🌟 Real-Life Analogy
Tailoring a suit
Underfitting = one-size-fits-all suit — fits nobody perfectly.
Good fit = tailored suit — fits this person's measurements well.
Overfitting = suit tailored to every crease and fold of ONE person's body — unusable for anyone else!

🐍 Comparing Degrees in Python
from sklearn.metrics import mean_squared_error, r2_score from sklearn.model_selection import train_test_split from sklearn.preprocessing import PolynomialFeatures from sklearn.linear_model import LinearRegression X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2) for deg in [1, 2, 3, 5, 9]: poly = PolynomialFeatures(degree=deg) X_tr = poly.fit_transform(X_train) X_te = poly.transform(X_test) m = LinearRegression().fit(X_tr, y_train) print(f"Deg {deg} | Train R²: {r2_score(y_train, m.predict(X_tr)):.3f}" f" | Test R²: {r2_score(y_test, m.predict(X_te)):.3f}")

🧪 Concept 3

sklearn Pipeline

The right way to do polynomial regression: chain PolynomialFeatures → LinearRegression in a single reusable Pipeline.

🔗
Why Pipeline? It prevents data leakage, makes code cleaner, and lets you use cross-validation and GridSearchCV seamlessly — transform and fit always stay in sync.

🌟 Real-Life Analogy
Factory assembly line
A Pipeline is like an assembly line in a factory. Raw material (data) enters one end, passes through each station (transform, fit), and finished product (predictions) comes out the other. Every station must be set up before it can process. You can't skip or reorder stations — that's exactly what Pipeline enforces!
▶ Building a Polynomial Pipeline
from sklearn.pipeline import Pipeline from sklearn.preprocessing import PolynomialFeatures from sklearn.linear_model import LinearRegression # Create the pipeline pipe = Pipeline([ ('poly', PolynomialFeatures(degree=3)), ('lr', LinearRegression()) ]) # Train the model pipe.fit(X_train, y_train) # Make predictions y_pred = pipe.predict(X_test)
🐍 Full Pipeline + Cross Validation
import numpy as np from sklearn.pipeline import Pipeline from sklearn.preprocessing import PolynomialFeatures, StandardScaler from sklearn.linear_model import LinearRegression from sklearn.model_selection import cross_val_score # Build the pipeline pipe = Pipeline([ ('poly', PolynomialFeatures(degree=3, include_bias=False)), ('scaler', StandardScaler()), # important for stability! ('lr', LinearRegression()) ]) # Cross-validate (5-fold) scores = cross_val_score(pipe, X, y, cv=5, scoring='r2') print(f"CV R² scores: {scores}") print(f"Mean R²: {scores.mean():.3f} ± {scores.std():.3f}") # Fit and predict pipe.fit(X_train, y_train) y_pred = pipe.predict(X_test)

Predictions: [174.5, 223.1, 156.8, 201.4, 189.2]
# Consistent across folds → model generalises well!
⚖️ Advanced Concept

Bias-Variance Tradeoff

The fundamental tension in machine learning: reducing bias increases variance and vice versa. Polynomial degree is the knob that controls this balance.

Student learning strategies

🐍 Fixing Overfitting with Ridge Regularisation
from sklearn.linear_model import Ridge from sklearn.pipeline import Pipeline from sklearn.preprocessing import PolynomialFeatures, StandardScaler # High degree but REGULARISED — avoids overfitting ridge_pipe = Pipeline([ ('poly', PolynomialFeatures(degree=9)), ('scaler', StandardScaler()), ('ridge', Ridge(alpha=10.0)) # alpha penalises large coefficients ]) ridge_pipe.fit(X_train, y_train) print(f"Train R²: {ridge_pipe.score(X_train, y_train):.3f}") print(f"Test R²: {ridge_pipe.score(X_test, y_test):.3f}")
Train R²: 0.968
Test R²: 0.941
# Ridge with degree 9 ≈ degree 3 performance — regularisation works!

Fit a Quadratic Model
Given the dataset below, fit a polynomial regression model of degree 2 and predict the output for x = 6.5. Print the R² score on training data.

import numpy as np from sklearn.preprocessing import PolynomialFeatures from sklearn.linear_model import LinearRegression from sklearn.metrics import r2_score X = np.array([[1],[2],[3],[4],[5],[6],[7],[8]]) y = np.array([2,8,18,32,50,72,98,128]) poly = PolynomialFeatures(degree=2) X_p = poly.fit_transform(X) model = LinearRegression().fit(X_p, y) pred = model.predict(poly.transform([[6.5]])) print(f"Prediction at 6.5: {pred[0]:.2f}") print(f"R² score: {r2_score(y, model.predict(X_p)):.4f}")
2
Build a Full Pipeline with Cross-Validation
Create a polynomial regression Pipeline (degree=4 with StandardScaler) and evaluate it using 5-fold cross-validation. Print mean R² and standard deviation. Use the Boston-like dataset with:

np.random.seed(42); X = np.random.rand(100,1)*10; y = 3*X**2 - 2*X + np.random.randn(100,1)*5
import numpy as np from sklearn.pipeline import Pipeline from sklearn.preprocessing import PolynomialFeatures, StandardScaler from sklearn.linear_model import LinearRegression from sklearn.model_selection import cross_val_score np.random.seed(42) X = np.random.rand(100,1)*10 y = (3*X**2 - 2*X + np.random.randn(100,1)*5).ravel() pipe = Pipeline([ ('poly', PolynomialFeatures(degree=4)), ('scaler', StandardScaler()), ('lr', LinearRegression()) ]) scores = cross_val_score(pipe, X, y, cv=5, scoring='r2') print(f"Mean R²: {scores.mean():.3f} ± {scores.std():.3f}")
3
GridSearchCV — Find Optimal Degree
Use GridSearchCV to automatically find the best polynomial degree from [1, 2, 3, 4, 5] for a dataset. Print the best degree and the best cross-validated R² score.
from sklearn.model_selection import GridSearchCV from sklearn.pipeline import Pipeline from sklearn.preprocessing import PolynomialFeatures, StandardScaler from sklearn.linear_model import LinearRegression pipe = Pipeline([ ('poly', PolynomialFeatures()), ('scaler', StandardScaler()), ('lr', LinearRegression()) ]) param_grid = {'poly__degree': [1,2,3,4,5]} grid = GridSearchCV(pipe, param_grid, cv=5, scoring='r2') grid.fit(X, y) print(f"Best degree: {grid.best_params_['poly__degree']}") print(f"Best CV R²: {grid.best_score_:.3f}")
