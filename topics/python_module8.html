🪜 Step-by-Step: How open() Works

1
Python locates the file on disk
Python checks the specified path. If the file doesn't exist and mode is 'r', it raises a FileNotFoundError.
2
OS grants access based on mode
The operating system opens a file descriptor — a low-level handle that points to the file's location on disk.
3
A file object is returned
Python wraps the file descriptor in a convenient file object with methods like .read(), .write(), .close().
4
Always call .close() when done
Closing releases the OS file lock and flushes any buffered writes to disk. Forgetting to close can corrupt files or cause resource leaks.

The with Statement 🔒
The with statement creates a context manager that automatically closes the file — even if an error occurs. It's the Python-recommended way to handle files.
🏧
Real-Life Analogy

Think of an ATM transaction. When you insert your card, the machine "opens" a session. When you're done, it automatically closes the session whether the transaction succeeded or not. The with statement works exactly the same way for files!

⚖️ Without vs With — Visual Comparison

❌ The Risky Way
f = open("data.txt", "r") content = f.read() # ⚠️ If error happens here... f.close() # ...this never runs! → File stays locked 🔴
VS
✅ The Safe Way
with open("data.txt", "r") as f: content = f.read() # ✅ Error or not... → File auto-closes 🟢

💻 Code Example

Python 1# Reading with context manager 2with open("students.txt", "r") as f: 3 for line in f: 4 print(line.strip()) 5 6# Writing with context manager 7with open("output.txt", "w") as f: 8 f.write("Data saved!\n")
>>> Alice >>> Bob >>> Charlie >>> File auto-closed ✓

🔄 The with Statement Flow

Here's exactly what happens when Python runs a with block:

1. open("file.txt") called
2. __enter__() → file object assigned to f
3. Code block runs
4. __exit__() → f.close() auto-called
5. File safely closed ✓

Real-world data often lives in CSV (spreadsheets) or JSON (APIs) format. Python has built-in modules to handle both effortlessly.

📋

🌐 JSON Files

🗂️
Analogy

JSON = Python dictionary saved as text. Used by every web API to send/receive data.

Coding Exercises 🧪
Try solving each problem first, then reveal the solution to check your approach.
1
Count lines in a file using the with statement
open() · read

Write a Python program that opens "story.txt" and prints the total number of lines it contains.

with open("story.txt", "r") as f: lines = f.readlines() print(f"Total lines: {len(lines)}") # Output: Total lines: 42
2
Append a new entry to a log file
write · append

Write a function log_event(message) that appends a timestamped message to "app.log". Each call should add a new line, not overwrite.

from datetime import datetime def log_event(message): timestamp = datetime.now().strftime("%Y-%m-%d %H:%M") with open("app.log", "a") as f: f.write(f"[{timestamp}] {message}\n") log_event("User logged in") log_event("File uploaded") # app.log now has 2 new lines!
3
Read a JSON config and update a value
JSON · dict

Given a config.json with {"theme": "light", "version": 1}, read it, change the theme to "dark", and write it back.

import json # Step 1: Read existing config with open("config.json", "r") as f: config = json.load(f) # Step 2: Modify config["theme"] = "dark" # Step 3: Write back with open("config.json", "w") as f: json.dump(config, f, indent=2) print("Config updated!")
