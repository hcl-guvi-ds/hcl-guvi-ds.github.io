Bayes' Theorem — The Foundation 🎲
Bayes' Theorem lets us update our belief about an event given new evidence. It's the backbone of Naive Bayes and probabilistic reasoning in AI.
☂️
Real-Life Analogy

Weather forecaster with new evidence: You think it'll rain with 40% probability (prior). Then you look outside and see dark clouds (evidence). You update your belief to 85% rain probability (posterior). Bayes' Theorem is this exact process — mathematically formalised: P(Rain|Clouds) ∝ P(Clouds|Rain) × P(Rain).

💻 Bayes' Theorem in Python

# P(Disease|+Test) using Bayes' Theorem P_disease = 0.10 # Prior: 10% prevalence P_test_given_D = 0.90 # Sensitivity: 90% P_test_given_ND = 0.05# False positive: 5% # P(+Test) = total probability P_test = (P_test_given_D * P_disease + P_test_given_ND * (1 - P_disease)) # Bayes' Theorem P_D_given_test = (P_test_given_D * P_disease / P_test) print(f"P(+Test): {P_test:.3f}") print(f"P(Disease|+Test): {P_D_given_test:.3f}")

🪜 How Naive Bayes Classifies

1
Start with prior probabilities
For each class, calculate P(class) from training data. If 60% emails are spam, P(spam)=0.6. This is your baseline belief before seeing any features.
2
Calculate likelihoods for each feature
For each word/feature, compute P(word|class). How often does "free" appear in spam? In ham? These are the likelihood tables built from training data.
3
Multiply likelihoods × prior (naively!)
Assume all features are independent (the "naive" part) and multiply: P(spam) × P(free|spam) × P(offer|spam) × ... This gives an unnormalised score for each class.
4
Pick the class with highest score
Normalise scores to get probabilities. The class with the highest posterior probability wins. In practice, we use log-probabilities to prevent numerical underflow.

Naive Bayes Spam Detector 📧
The classic NB application: classify emails as spam or ham (not spam) based on the words they contain. Each word is a "feature" — we compute its probability given each class.
🕵️
Real-Life Analogy

Like a detective reading clues: "free", "winner", "urgent" → very suspicious (high P(word|spam)). "meeting", "project", "report" → very normal (high P(word|ham)). The detective (NB) multiplies all clue probabilities together and picks the most likely suspect. Simple, fast, and surprisingly accurate!

💻 NB Spam Classifier

from sklearn.naive_bayes import MultinomialNB from sklearn.feature_extraction.text import CountVectorizer from sklearn.pipeline import Pipeline emails = [ "free prize winner urgent", "project meeting schedule", "click offer cash guaranteed", "please review the report", ] labels = [1, 0, 1, 0] # 1=spam, 0=ham pipeline = Pipeline([ ('vectorizer', CountVectorizer()), ('classifier', MultinomialNB()) ]) pipeline.fit(emails, labels) test = ["you are a winner free cash"] pred = pipeline.predict_proba(test) print(f"P(Ham): {pred[0][0]:.4f}") print(f"P(Spam): {pred[0][1]:.4f}")

Three Types of Naive Bayes 📊
Naive Bayes comes in three flavours, each designed for a different type of feature distribution. Choosing the right one dramatically impacts accuracy.
🎭
Real-Life Analogy

Three different measurement tools: GaussianNB is like a thermometer (measures continuous values with a bell curve). MultinomialNB is like a vote counter (counts how many times each word appears). BernoulliNB is like a Yes/No checklist (did this word appear or not?). Use the right tool for your data type.

🧩 NB Type Selection Guide — Click to Reveal

📈 GaussianNB
Use when features are continuous and roughly bell-curve distributed. Examples: height, weight, temperature, sensor readings, income. It estimates μ and σ² for each class. Works great with the Iris dataset.
👆 Click to reveal
📝 MultinomialNB
Use for count data — how many times a word appears in a document. Best for text classification with word frequencies (TF or TF-IDF). Requires non-negative features. Default for NLP tasks.
👆 Click to reveal
☑️ BernoulliNB
Use for binary features — did this word appear (1) or not (0)? Penalises absence of terms unlike MultinomialNB. Better for short documents. Good for sentiment analysis with binary bag-of-words.
👆 Click to reveal
⚡ Why "Naive"?
The algorithm assumes conditional independence between features given the class. In reality, "free" and "winner" are correlated in spam emails, but NB ignores this. Despite being wrong, this assumption works surprisingly well in practice!

Advanced Concept
Laplace Smoothing & NB Limitations ⚡
What happens when a word never appeared in training? The probability collapses to 0, killing the entire prediction. Laplace smoothing solves this elegantly — but NB still has real limitations to know.

from sklearn.naive_bayes import MultinomialNB # var_smoothing (GaussianNB) or alpha (MultinomialNB) # Default alpha=1.0 (Laplace smoothing) nb1 = MultinomialNB(alpha=1.0) # Lidstone smoothing with alpha=0.5 nb2 = MultinomialNB(alpha=0.5) # No smoothing (dangerous!) nb3 = MultinomialNB(alpha=0.0) # GaussianNB uses var_smoothing from sklearn.naive_bayes import GaussianNB gnb = GaussianNB(var_smoothing=1e-9) # The formula: P(w|c) = (count+α)/(N_c + α×V) # where V = vocabulary size print("Default alpha=1 is usually best")

🌍 Real-World Scenarios — Click to Explore

🏥
Medical Diagnosis
GaussianNB works well for continuous medical measurements (BP, glucose). Fast enough for real-time clinical decision support. The independence assumption is often violated but accuracy remains acceptable.
📰
News Classification
MultinomialNB + TF-IDF is the gold standard for text classification. Sports vs Politics vs Tech news can be classified with 95%+ accuracy. Trains in milliseconds on millions of articles.
⚡
Real-Time Filtering
NB classifies 10,000+ emails per second — impossible for neural networks in real-time. Zero infrastructure needed, perfect for embedded systems and edge computing with limited resources.
⚠️ Known Limitations
🚫 Independence assumption — features are rarely truly independent in real data
📊 Poor calibration — probability estimates are often overconfident (too extreme)
🔢 Continuous correlations — can't capture feature interactions like XGBoost can

from sklearn.naive_bayes import GaussianNB from sklearn.datasets import load_iris from sklearn.model_selection import cross_val_score, train_test_split import numpy as np X, y = load_iris(return_X_y=True) names = load_iris().target_names gnb = GaussianNB() scores = cross_val_score(gnb, X, y, cv=10) print(f"CV Accuracy: {scores.mean():.4f} ± {scores.std():.4f}") gnb.fit(*train_test_split(X, y, test_size=0.2)[:2]) print("\nLearned means per class:") for i, name in enumerate(names): print(f"{name}: {gnb.theta_[i].round(2)}") samples = np.array([[5.1,3.5,1.4,0.2], [6.3,3.3,4.7,1.6], [6.5,3.0,5.2,2.0]]) preds = gnb.predict(samples) print("\nCustom predictions:", names[preds])

from sklearn.datasets import fetch_20newsgroups from sklearn.naive_bayes import GaussianNB, MultinomialNB, BernoulliNB from sklearn.feature_extraction.text import TfidfVectorizer, CountVectorizer from sklearn.pipeline import Pipeline from sklearn.model_selection import cross_val_score import numpy as np data = fetch_20newsgroups(subset='all', categories=[ 'sci.space', 'rec.sport.hockey', 'talk.politics.guns' ]) X, y = data.data, data.target pipelines = { 'MultinomialNB': Pipeline([ ('cv', CountVectorizer()), ('nb', MultinomialNB())]), 'BernoulliNB': Pipeline([ ('cv', CountVectorizer(binary=True)), ('nb', BernoulliNB())]), } for name, pipe in pipelines.items(): s = cross_val_score(pipe, X, y, cv=3) print(f"{name}: {s.mean():.4f}")
