Why Transformers Matter Big Picture

Before Transformers, sequence models usually processed text one step at a time. That made long-range relationships hard and training slow. Transformers changed the recipe: instead of reading left-to-right with a recurrent hidden state, they compare tokens directly with attention. This makes training parallel, captures long-distance dependencies, and scales beautifully with data and compute.

AI

Analogy: Imagine reading a paragraph with sticky notes. For each word, you ask: "Which other words explain this word?" In "The animal did not cross the road because it was tired", attention helps decide that "it" refers to "animal", not "road".

Self-Attention Lab

Self-attention lets each token build a context-aware representation by borrowing information from other tokens. This is the beating heart of Transformers.

Attention Formula Scaled Dot Product
Attention(Q,K,V) = softmax(QKT / sqrt(dk)) V
Query (Q): what the current token is looking for.
Key (K): what each token offers as a searchable label.
Value (V): the actual information copied into the output.

Transformer Architecture

A Transformer block repeats a few operations: attention, residual connections, normalization, and a feed-forward network. Stacking many blocks creates powerful representations.

Causal Mask Demo Decoder Logic

In generation, token 5 must not peek at token 6. A causal mask hides future positions so next-token prediction remains honest.

Mask: ON
PyTorch-style Transformer block
class TransformerBlock(nn.Module):
    def forward(self, x, mask=None):
        attn_out = self.attention(x, x, x, mask)
        x = self.norm1(x + attn_out)       # residual + layer norm
        mlp_out = self.mlp(x)
        x = self.norm2(x + mlp_out)        # another residual path
        return x

Training Stages
StageWhat Happens
PretrainingPredict masked or next tokens from huge corpora. The model learns language, facts, and patterns.
Supervised fine-tuningTrain on input-output examples so the model follows desired task formats.
Preference tuningUse rankings or reward signals to prefer helpful, harmless, high-quality responses.
InferenceGenerate tokens one at a time using sampling controls like temperature and top-p.
Important Failure Modes
Hallucination: fluent output that is not grounded in truth.
Context limits: the model can only attend to tokens inside its context window.
Data leakage: evaluation becomes misleading if test examples appear in training.
Chapter 5

Try It Yourself

Use these hands-on mini tasks to check whether the ideas are becoming operational. Try solving before revealing the explanation.

1
Find the Attention Target
Sentence: "Maya placed the laptop on the desk because it was charging." What should "it" attend to?
"It" should most strongly attend to "laptop", because a laptop can be charging. If it attended to "desk", the meaning would become strange. This is a coreference problem, and attention heads often learn patterns that help resolve it.
2
Choose the Model Type
You need a model for sentiment classification of reviews. Encoder-only, decoder-only, or encoder-decoder?
Encoder-only is usually the most direct choice. The full review can be read bidirectionally, then a classification head predicts sentiment. Decoder-only can also classify via prompting, but encoder-only is efficient and natural for fixed-label prediction.
3
Causal Mask Reasoning
During next-token prediction, why must the model not see future tokens?
If the model sees future tokens, the training task becomes cheating. It could copy the answer instead of learning how previous context predicts the next token. Causal masking ensures training matches generation, where future tokens do not exist yet.
