Did you know? Python has over 60 built-in exception types. The most common ones are ValueError, TypeError, ZeroDivisionError, and FileNotFoundError.
📘 Concept 1
What are Exceptions?
An exception is an error that occurs during program execution — a runtime error that crashes your program if unhandled.
🚗
Real-Life Analogy: Dashboard Warnings
When something goes wrong while driving, your dashboard shows a warning light. Python exceptions are like those warnings — without handling them, the car stalls immediately. With handling, you can respond gracefully.

📘 Concept 2
try / except
Wrap risky code in a try block. If an exception occurs, Python jumps to the matching except block — your program keeps running!
🏥
Real-Life Analogy: Emergency Plan
Try: Attempt the normal procedure. Except: If something goes wrong, activate the emergency plan. The hospital (your program) doesn't shut down — it handles the situation and continues.

📘 Concept 3
else & finally
else runs only if no exception occurred; finally runs no matter what — even if there's an exception or a return statement.

✅ else block

Runs only when the try block succeeds with no exceptions. Perfect for success-only code.

🔒 finally block

Runs always — success or failure. Perfect for cleanup: closing files, releasing resources.

else_finally.py
try:
    result = 10 / b
except ZeroDivisionError:
    print("Cannot divide by zero!")
else:
    print(f"Result: {result}")
finally:
    print("Cleanup complete")
Cannot divide by zero!

📋
Order must be: try → except → else → finally. You can skip else or finally, but order cannot be changed.
⭐ Advanced Concept
Custom Exceptions
Define your own exception types by creating a class that inherits from Exception — powerful for domain-specific error handling.
🏦
Real-Life Analogy: Bank Custom Rules
A bank doesn't use a generic "error" when you overdraft. It uses InsufficientFundsError with the amount short. Custom exceptions give your errors context and meaning.
Exception name
Message
custom_exceptions.py
class InsufficientFundsError(Exception):
    def __init__(self, balance, amount):
        super().__init__(f"Need ₹{amount}, only ₹{balance} available")

def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientFundsError(balance, amount)
    return balance - amount
Transaction failed: Need ₹800, only ₹500 available
⬆️
The raise keyword lets you throw exceptions manually. Use it to validate inputs and signal errors in your own code.

⭐ Exercise 1
Safe Division Calculator
Write a function safe_divide(a, b) that handles ZeroDivisionError and TypeError. Return the result or error message.
def safe_divide(a, b):
    try:
        return a / b
    except ZeroDivisionError:
        return "Cannot divide by zero!"
    except TypeError:
        return "Both inputs must be numbers"
⭐ Exercise 2
Safe File Reader
Write a function read_file(filename) that handles FileNotFoundError and PermissionError. Use finally to print "File operation complete."
def read_file(filename):
    try:
        with open(filename, 'r') as f:
            print(f.read())
    except FileNotFoundError:
        print(f"File '{filename}' not found")
    except PermissionError:
        print("Permission denied")
    finally:
        print("File operation complete.")
⭐ Exercise 3
Custom AgeValidationError
Create a custom exception AgeValidationError that inherits from ValueError. Write a function that raises it for invalid ages.
class AgeValidationError(ValueError):
    def __init__(self, age, reason):
        super().__init__(f"Invalid age {age}: {reason}")

def validate_age(age):
    if not isinstance(age, int):
        raise AgeValidationError(age, "must be integer")
    if age < 0 or age > 130:
        raise AgeValidationError(age, "out of range")
    return age
