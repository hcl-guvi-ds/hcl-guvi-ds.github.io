Concept 1

What are Exceptions?

An exception is an error that occurs during program execution — not a syntax error (caught before running), but a runtime error that crashes your program if unhandled.

🚗

Real-Life Analogy: Car Dashboard Warnings

When something goes wrong while driving, your dashboard shows a warning light — it doesn't stop the car immediately. Python exceptions are like those warnings. Without exception handling, the car immediately stalls. With it, you can respond gracefully: pull over, check the issue, and continue.

Concept 2

try / except

Wrap risky code in a try block. If an exception occurs, Python jumps to the matching except block — your program keeps running!

🏥

Real-Life Analogy: Hospital Emergency Plan

Try: Attempt the normal procedure. Except: If something goes wrong, activate the emergency plan. The hospital (your program) doesn't shut down — it gracefully handles the situation and continues operating.

try_except.py
# Catch a specific exception
try:
    age = int(input("Enter age: "))  # risky!
    print(f"Your age is {age}")
except ValueError:
    print("Please enter a valid number!")

# Catch multiple exceptions
try:
    result = 10 / int(input("Divisor: "))
    print(f"Result: {result}")
except ValueError:
    print("Not a number!")
except ZeroDivisionError:
    print("Can't divide by zero!")
except Exception as e:
    print(f"Unexpected error: {e}")

Concept 3

else & finally

Two powerful additions to the try/except block: else runs only if no exception occurred; finally runs no matter what — even if there's an exception or a return statement.

✅ else block

Runs only when the try block succeeds with no exceptions. Use it for code that should only run on success.

🔒 finally block

Runs always — success or failure. Perfect for cleanup: closing files, releasing resources, database disconnects.

else_finally.py
def safe_divide(a, b):
    try:
        result = a / b
    except ZeroDivisionError:
        print("❌ Cannot divide by zero!")
    else:
        print(f"✅ Result: {result}")  # only if no exception
    finally:
        print("🔒 Always runs — cleanup done")

safe_divide(10, 2)
safe_divide(10, 0)

📋 Complete try/except/else/finally Structure

try:                # attempt risky code
    ...code...
except ValueError:     # specific exception
    ...handle...
except Exception as e: # catch-all
    ...handle...
else:                 # no exception? run this
    ...success code...
finally:             # ALWAYS runs
    ...cleanup...
The order must always be: try → except → else → finally. You can skip else or finally, but you can never change the order.
Advanced Concept

Custom Exceptions

You can define your own exception types by creating a class that inherits from Exception. This is powerful for domain-specific error handling.

🏦

Real-Life Analogy: Bank Custom Rules

A bank doesn't use a generic "error" when you overdraft. It uses a specific InsufficientFundsError with the amount short. Custom exceptions give your errors context, meaning, and precise handling.

custom_exceptions.py
# Define a custom exception
class InsufficientFundsError(Exception):
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(
            f"Need ₹{amount}, but only ₹{balance} available"
        )

# Use it in your code
def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientFundsError(balance, amount)
    return balance - amount

try:
    withdraw(500, 800)
except InsufficientFundsError as e:
    print(f"Transaction failed: {e}")

⬆️ The raise keyword

• raise — re-raises the current exception (inside except)
• raise ValueError("msg") — raise a specific exception manually
• raise InsufficientFundsError(500, 800) — raise custom exception
raise_examples.py
# Validate input and raise if invalid
def set_age(age):
    if not isinstance(age, int):
        raise TypeError(f"Age must be int, got {type(age).__name__}")
    if age < 0 or age > 150:
        raise ValueError(f"Age {age} is not realistic")
    return age

print(set_age(25))           # 25
print(set_age("twenty"))   # TypeError

Exercise 01Safe Division Calculator

Write a function safe_divide(a, b) that handles both ZeroDivisionError and TypeError. Return the result if successful, or a helpful error message string if not. Test with (10, 2), (10, 0), and (10, "five").

Use multiple except blocks and return different messages for each.
solution.py
def safe_divide(a, b):
    try:
        result = a / b
    except ZeroDivisionError:
        return "Error: Cannot divide by zero!"
    except TypeError:
        return f"Error: Both inputs must be numbers, got {type(b).__name__}"
    else:
        return f"Result: {result}"
    finally:
        print("Division attempted.")

print(safe_divide(10, 2))      # Result: 5.0
print(safe_divide(10, 0))      # Error: Cannot divide by zero!
print(safe_divide(10, "five")) # Error: Both inputs must be numbers, got str
Exercise 02Safe File Reader

Write a function read_file(filename) that tries to open and read a file. Handle FileNotFoundError and PermissionError separately. Use finally to print "File operation complete." regardless.

solution.py
def read_file(filename):
    try:
        with open(filename, 'r') as f:
            content = f.read()
            print(f"File contents:\n{content}")
    except FileNotFoundError:
        print(f"❌ File '{filename}' does not exist.")
    except PermissionError:
        print(f"❌ No permission to read '{filename}'.")
    else:
        print("✅ File read successfully!")
    finally:
        print("📁 File operation complete.")

read_file("data.txt")       # if exists, reads it
read_file("ghost.txt")      # FileNotFoundError
Exercise 03Custom AgeValidationError

Create a custom exception AgeValidationError that inherits from ValueError. Write a function validate_age(age) that raises this exception if age is negative, non-integer, or greater than 130. Include the invalid value in the error message.

solution.py
class AgeValidationError(ValueError):
    def __init__(self, age, reason):
        self.age = age
        super().__init__(f"Invalid age {age!r}: {reason}")

def validate_age(age):
    if not isinstance(age, int):
        raise AgeValidationError(age, "must be an integer")
    if age < 0:
        raise AgeValidationError(age, "cannot be negative")
    if age > 130:
        raise AgeValidationError(age, "unrealistically large")
    return f"Valid age: {age}"

try:
    validate_age(-5)
except AgeValidationError as e:
    print(f"Caught: {e}")
