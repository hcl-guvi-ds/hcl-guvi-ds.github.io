📐 Topic 1 of 8

Descriptive Statistics

Summarise and understand your data before modelling. Descriptive stats tell you WHERE the data centres (central tendency) and HOW SPREAD it is (dispersion).

📏

Analogy — The Class Report Card: Imagine 20 students' exam scores. The mean is the class average. The median is the middle student's score (not affected by one genius or one failing student). The mode is the most common score. The standard deviation tells you if everyone scored similarly or if scores were all over the place.

Distribution of Values
💻 Descriptive Stats in Python Code
python
import numpy as np
import pandas as pd
from scipy import stats

data = [12,18,24,18,30,15,22,18,28,35]

print(f"Mean:     {np.mean(data):.2f}")

print(f"Range:    {np.max(data)-np.min(data)}")
print(f"IQR:      {np.percentile(data,75)-np.percentile(data,25):.2f}")
print(f"Skewness: {stats.skew(data):.3f}")
print(f"Kurtosis: {stats.kurtosis(data):.3f}")

# Pandas describe gives everything at once
s = pd.Series(data)
print(s.describe())

🔍 Measures Explained
📊
Mean
Click to reveal
Average of all values. Sum(x)/n. Sensitive to outliers — one extreme value pulls it strongly. Best for symmetric distributions without outliers.
🎯
Median
Click to reveal
Middle value when sorted. Robust to outliers — that's why house prices use median income, not mean income. Perfect for skewed data.
👑
Mode
Click to reveal
Most frequent value. Only central tendency for categorical data (e.g., most common colour). Can be bimodal (two peaks) or multimodal.
📏
Std Dev
Click to reveal
Average distance from mean. sigma = sqrt(Var). The 68-95-99.7 rule: 68% of normal data falls within 1 std dev of mean. Use ddof=1 for samples!
📦
IQR
Click to reveal
Interquartile Range = Q3 - Q1. Middle 50% of data. Outlier rule: any value beyond Q1 - 1.5*IQR or Q3 + 1.5*IQR. Robust like median.
↗️
Skewness
Click to reveal
Shape asymmetry. Right skew (positive): long tail right, mean > median (income, salary). Left skew (negative): long tail left, mean < median (exam scores at top).

Probability Distributions

A distribution describes all possible values and their probabilities. Different real-world phenomena follow different distributions — knowing which to use is a superpower.

🎰

Analogy — Different types of random events: Flipping a coin 10 times → Binomial. Number of customers arriving per hour → Poisson. Adult heights or exam scores → Normal (Gaussian). Time until a machine breaks → Exponential. Each has a different shape, mean, and variance formula.

🎮 Distribution Visualiser Interactive
💻 Working with Distributions
python
from scipy import stats
import numpy as np

# Normal Distribution
norm = stats.norm(loc=0, scale=1)
print("P(X < 1.96):", norm.cdf(1.96))  # 0.975
print("95th percentile:", norm.ppf(0.95))

# Binomial: 10 flips, P(H)=0.5
binom = stats.binom(n=10, p=0.5)
print("P(X=6):", binom.pmf(6))  # 0.205
print("P(X≥8):", 1-binom.cdf(7))

# Poisson: avg 3 arrivals/hour
pois = stats.poisson(mu=3)
print("P(X=5):", pois.pmf(5))

# Sample from any distribution
samples = norm.rvs(size=1000)
print(f"Sample mean: {samples.mean():.3f}")
>>>P(X < 1.96): 0.9750 ← 97.5% of data below z=1.96
>>>95th percentile: 1.6449
>>>P(X=6 heads): 0.2051
>>>P(X≥8 heads): 0.0547
>>>P(5 arrivals|avg=3): 0.1008
68-95-99.7 Rule: within 1/2/3 std devs of normal mean!

Probability Theory

Probability is the mathematical language of uncertainty. Every machine learning prediction, every confidence interval, every hypothesis test is built on probability foundations.

🎯

Analogy — Weather forecasting: "70% chance of rain" means if we could replay today's conditions 100 times, it rains in 70 of them. Probability is not about a single event — it's about frequencies over many repetitions. This frequentist view underpins most classical statistics.

📜 The Rules of Probability
1
Addition Rule: P(A or B)
P(A∪B) = P(A) + P(B) - P(A∩B). Subtract the overlap to avoid double-counting. If mutually exclusive: P(A∪B) = P(A)+P(B).
2
Multiplication Rule: P(A and B)
P(A∩B) = P(A) × P(B|A). If independent: P(A∩B) = P(A) × P(B). Independence means knowing A tells us nothing about B.
3
Conditional Probability: P(A|B)
P(A|B) = P(A∩B)/P(B). "Probability of A given B has occurred." Restricts the sample space to only cases where B is true.
4
Complement Rule: P(not A)
P(A') = 1 - P(A). The probability of A NOT occurring. Often easier to compute P(A') and subtract from 1.
5
Bayes' Theorem: Updating beliefs
P(A|B) = P(B|A)×P(A) / P(B). Update your prior belief about A with evidence B. Foundation of Bayesian statistics and spam filters!
💻 Probability Simulations
python
import numpy as np

# Law of Large Numbers simulation
np.random.seed(42)
flips = np.random.choice([0,1], size=10000)
running_avg = np.cumsum(flips) / np.arange(1,10001)

print(f"After 10 flips:    {running_avg[9]:.3f}")
print(f"After 100 flips:   {running_avg[99]:.3f}")
print(f"After 1000 flips:  {running_avg[999]:.3f}")
print(f"After 10000 flips: {running_avg[9999]:.3f}")
print(f"True probability:  0.500")

# Birthday paradox
def birthday_prob(n):
    p = 1.0
    for i in range(n):
        p *= (365-i)/365
    return 1-p

for n in [10,23,30,50]:
    print(f"n={n:2d}: P(shared bday)={birthday_prob(n):.3f}")

>>>After 1000 flips: 0.498
>>>After 10000 flips: 0.501 ← converges to 0.5!

>>>n=23: P(shared bday)=0.507 ← >50% with just 23 people!

Correlation & Covariance

Correlation measures the strength and direction of a linear relationship between two variables. It NEVER implies causation — one of statistics' most important lessons.

🍦

Famous example — Ice cream and drowning: Ice cream sales and drowning deaths are strongly correlated (r ≈ 0.85). Does ice cream cause drowning? No! Both are caused by hot weather — a confounding variable. Correlation finds patterns; causation requires controlled experiments. Always ask "what could cause both?"

💻 Correlation in Python
python
import numpy as np
import pandas as pd
from scipy.stats import pearsonr, spearmanr

x = [2,4,5,7,8,9,10,12]
y = [3,6,5,9,11,10,13,15]

r, p = pearsonr(x, y)
rho, p2 = spearmanr(x, y)

print(f"Pearson r:  {r:.4f}  p={p:.4f}")
print(f"Spearman:  {rho:.4f}  p={p2:.4f}")
print(f"R² (variance explained): {r**2:.4f}")

# Full correlation matrix
df = pd.DataFrame({'x':x, 'y':y,
     'z':[1,3,2,4,6,5,8,7]})
print(df.corr())
>>>Pearson r: 0.9847 p=0.0000 ← Very strong, significant!
>>>Spearman: 0.9833 p=0.0000
>>>R² (variance explained): 0.9696 ← x explains 97% of y's variance
Correlation is significant (p<0.05) but NEVER implies x causes y!

Hypothesis Testing

Hypothesis testing uses data to make decisions under uncertainty. We start with a null hypothesis (status quo), collect evidence, and decide whether to reject it based on p-values.

⚖️

Analogy — Court trial: The null hypothesis (H₀) is "innocent until proven guilty." We need strong evidence to reject it. The p-value is "how surprising is this evidence if the defendant is innocent?" If p < 0.05 (very surprising), we convict (reject H₀). If p ≥ 0.05, we acquit (fail to reject). We never "prove" innocence — we just lack evidence against it.

🧪 Common Statistical Tests
📊
One-sample t-test
Click
When: Compare sample mean to known population mean. H₀: sample mean = μ₀. Example: Is our drug's effect different from the standard 120mg?
👥
Two-sample t-test
Click
When: Compare means of two independent groups. H₀: μ₁ = μ₂. Example: Do men and women score differently on the same test?
🔄
Paired t-test
Click
When: Compare before-and-after for the SAME subjects. H₀: mean difference = 0. Example: Did a training program improve scores?
📦
Chi-Square test
Click
When: Test independence of two categorical variables. H₀: variables are independent. Example: Is purchase behaviour independent of gender?
📈
ANOVA
Click
When: Compare means of 3+ groups simultaneously. H₀: all group means are equal. Example: Do 4 different teaching methods produce the same average scores?
🔗
F-test
Click
When: Compare variances of two populations. H₀: σ₁² = σ₂². Also used in ANOVA and regression to test overall model significance.
💻 t-test in Python
python
from scipy import stats
import numpy as np

np.random.seed(42)
# Group A: control, Group B: treatment
A = np.random.normal(50, 10, 30)
B = np.random.normal(55, 10, 30)

# Independent two-sample t-test
t, p = stats.ttest_ind(A, B)
print(f"t-statistic: {t:.4f}")
print(f"p-value:     {p:.4f}")
print(f"Reject H0:   {p < 0.05}")

# Chi-square test of independence
obs = [[50,30],[20,40]]  # contingency table
chi2, p2, dof, exp = stats.chi2_contingency(obs)
print(f"Chi2: {chi2:.3f}  p={p2:.4f}")

# One-way ANOVA: 3 groups
G1 = np.random.normal(50,8,20)
G2 = np.random.normal(55,8,20)
G3 = np.random.normal(58,8,20)
f, p3 = stats.f_oneway(G1, G2, G3)
print(f"ANOVA F: {f:.3f}  p={p3:.4f}")
>>>t-statistic: -2.1874
>>>p-value: 0.0322
>>>Reject H0: True ← Treatment is significantly different!
>>>Chi2: 10.417 p=0.0012 ← Variables are dependent
>>>ANOVA F: 4.826 p=0.0108 ← At least one group differs

Central Limit Theorem & Confidence Intervals

The CLT is arguably the most important theorem in statistics. It explains why normal distributions appear everywhere and makes statistical inference possible.

🍪

Analogy — Cookie factory: Each cookie has a slightly different weight (any distribution). Grab a sample of 30 cookies and compute the average weight. Repeat this 1000 times. The distribution of these 1000 sample means will be normally distributed, regardless of the original cookie weight distribution. Magic? No — it's the Central Limit Theorem!

💻 CLT & Confidence Intervals
python
import numpy as np
from scipy import stats

# CLT demonstration
np.random.seed(42)
pop = np.random.exponential(scale=2, size=100000)  # skewed
print(f"Population: mean={pop.mean():.2f} skew={stats.skew(pop):.2f}")

# Take 5000 sample means
sample_means = [np.mean(np.random.choice(pop,30)) for _ in range(5000)]
print(f"Means skew: {stats.skew(sample_means):.3f} ← near 0!")

# 95% Confidence Interval
data = np.random.normal(100, 15, 50)
ci = stats.t.interval(
    confidence=0.95, df=len(data)-1,
    loc=np.mean(data),
    scale=stats.sem(data)
)
print(f"Sample mean: {data.mean():.2f}")
print(f"95% CI: [{ci[0]:.2f}, {ci[1]:.2f}]")
print(f"Margin of error: ±{(ci[1]-ci[0])/2:.2f}")
>>>Population: mean=2.00 skew=2.01 ← very skewed!
>>>Means skew: 0.013 ← nearly normal thanks to CLT!
>>>Sample mean: 99.84
>>>95% CI: [95.56, 104.12]
>>>Margin of error: ±4.28
We are 95% confident the true mean falls in [95.56, 104.12]

Bayesian Thinking

Bayesian statistics updates beliefs with evidence. Unlike frequentist statistics, it treats probability as a degree of belief, allowing you to incorporate prior knowledge into your analysis.

🔬

Medical test analogy: A rare disease affects 1% of people. A test is 95% accurate. You test positive. What's the probability you actually have the disease? Surprisingly only ~16%! The low base rate (prior) dominates. This is Bayes' Theorem in action: P(Disease|Positive) = P(Positive|Disease)×P(Disease) / P(Positive). Prior × Likelihood / Evidence = Posterior.

📐 The Three Pillars
1
Prior P(H)
Your initial belief before seeing data. Can be based on domain knowledge, historical data, or expert opinion. The prior can be "uninformative" (flat) if you want the data to speak for itself.
2
Likelihood P(Data|H)
How probable is the observed data given a particular hypothesis? This is where the new evidence comes in. The likelihood "weights" how much different priors should be updated.
3
Posterior P(H|Data)
Your updated belief after seeing the data. Posterior ∝ Prior × Likelihood. The posterior can become the prior for the next analysis — Bayesian learning is sequential!
4
Frequentist vs Bayesian
Frequentist: probability is long-run frequency. Bayesian: probability is degree of belief. Frequentist gives p-values. Bayesian gives "there's 95% probability the parameter is in this range" — more intuitive!
💻 Bayesian Updating
python
import numpy as np

# Bayesian coin flip: is coin fair?
# Prior: uniform (Beta(1,1))
# Observe 7 heads in 10 flips

# Beta-Binomial conjugate model
prior_a, prior_b = 1, 1  # uniform prior
heads, tails = 7, 3

# Posterior = Beta(a+heads, b+tails)
post_a = prior_a + heads
post_b = prior_b + tails

posterior_mean = post_a / (post_a + post_b)
# 95% Credible Interval
from scipy.stats import beta
lo, hi = beta.ppf([0.025,0.975], post_a, post_b)

print(f"Posterior mean: {posterior_mean:.3f}")
print(f"95% Credible Interval: [{lo:.3f}, {hi:.3f}]")
print(f"P(p>0.5 | data): {1-beta.cdf(0.5,post_a,post_b):.3f}")
>>>Posterior mean: 0.667 ← updated estimate of P(heads)
>>>95% Credible Interval: [0.381, 0.880]
>>>P(p>0.5 | data): 0.834 ← 83% chance coin is biased toward heads
More data → narrower credible interval → more certainty!

Regression Statistics

Regression statistics go beyond fitting a line — they tell you if the model is statistically significant, how much variance is explained, and which coefficients are trustworthy.

🎯

Analogy — Archery scorecard: R² is the overall score (how close to bullseye on average). Residual plots show whether your aiming is consistent (homoscedastic) or systematically off in certain directions. p-values for each feature tell you whether that particular technique actually improved your aim or just got lucky.

import statsmodels.api as sm

Hypothesis Testing — A/B Test Analysis
Simulate an A/B test: Group A (control) has click-through rate 10%, Group B (treatment) has 13%. With 1000 users each, run a two-proportion z-test. Report the p-value, whether the improvement is statistically significant at alpha=0.05, and the 95% confidence interval for the difference in proportions.
python
import numpy as np
from scipy import stats

np.random.seed(42)
n_a, n_b = 1000, 1000
p_a, p_b = 0.10, 0.13

clicks_a = np.random.binomial(n_a, p_a)
clicks_b = np.random.binomial(n_b, p_b)

p_a_hat = clicks_a / n_a
p_b_hat = clicks_b / n_b

# Pooled proportion
p_pool = (clicks_a + clicks_b) / (n_a + n_b)
se = np.sqrt(p_pool*(1-p_pool)*(1/n_a + 1/n_b))

z = (p_b_hat - p_a_hat) / se
p_val = 2 * (1 - stats.norm.cdf(abs(z)))

# 95% CI for difference
se_diff = np.sqrt(p_a_hat*(1-p_a_hat)/n_a + p_b_hat*(1-p_b_hat)/n_b)
diff = p_b_hat - p_a_hat
ci = (diff - 1.96*se_diff, diff + 1.96*se_diff)

print(f"Control CTR:   {p_a_hat:.3f}")
print(f"Treatment CTR: {p_b_hat:.3f}")
print(f"z-statistic:   {z:.3f}")
print(f"p-value:       {p_val:.4f}")
print(f"Significant:   {p_val < 0.05}")
print(f"95% CI diff:   [{ci[0]:.4f}, {ci[1]:.4f}]")
3
CLT Verification — Does it Work for All Distributions?
Verify the CLT empirically: generate a very skewed population (Chi-square with df=2). Take 10,000 samples of size n=5, n=30, n=100. For each n, compute the sample means and check: does skewness decrease? Does the distribution look more normal with larger n? Report skewness and kurtosis for each.
python
import numpy as np
from scipy import stats

np.random.seed(42)
# Very skewed population: Chi-square df=2
pop = stats.chi2.rvs(df=2, size=100000)
print(f"Population: skew={stats.skew(pop):.3f} kurt={stats.kurtosis(pop):.3f}")

for n in [5, 30, 100]:
    # 10000 sample means
    means = np.array([np.mean(np.random.choice(pop, n))
                      for _ in range(10000)])
    sk = stats.skew(means)
    ku = stats.kurtosis(means)
    sw_p = stats.shapiro(np.random.choice(means, 50)).pvalue
    print(f"n={n:3d}: skew={sk:.3f}  kurt={ku:.3f}  "
          f"Shapiro p={sw_p:.4f} {'(Normal!)' if sw_p>0.05 else ''}")
