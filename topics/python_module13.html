Why Algorithms Matter
Algorithms are the recipes of computing. They determine how fast your code runs, how much memory it uses, and whether it can scale to millions of users. Every great software engineer thinks algorithmically.
Concept 1 · Sorting
Sorting Algorithms
Arrange data in order — the foundation of efficient computation.

What is Bubble Sort?

Bubble Sort compares adjacent elements and swaps them if they're in the wrong order. It "bubbles" the largest element to the end in each pass. Simple but O(n²) — best for learning, not production!

🫧
Real-life analogy: Imagine sorting a queue of students by height. You walk down the line comparing each pair — if a taller student is before a shorter one, you swap them. After enough passes, everyone's in order. That's exactly how bubble sort works!

Python Code

bubble_sort.py
def bubble_sort(arr): n = len(arr) for i in range(n): # Last i elements are already sorted for j in range(0, n - i - 1): if arr[j] > arr[j+1]: # Swap adjacent elements arr[j], arr[j+1] = arr[j+1], arr[j] return arr data = [64, 34, 25, 12, 22, 11, 90] print(bubble_sort(data))
Output
[11, 12, 22, 25, 34, 64, 90]

Concept 2 · Searching
Searching Algorithms
Find elements efficiently — linear scan vs. smart binary splitting.
Linear Search
Binary Search

Linear Search

Check every element one by one from the start until you find the target (or exhaust the list). Works on unsorted data. Time complexity: O(n).

📚
Analogy: Searching for a name in an unsorted phone book — you start at page 1 and check every entry until you find it. Thorough but slow for large datasets.
linear_search.py
def linear_search(arr, target): for i, val in enumerate(arr): if val == target: return i # Found at index i return -1 # Not found nums = [4, 7, 2, 9, 1, 5] print(linear_search(nums, 9)) # 3
Output
3
# Found 9 at index 3

Binary Search

Divide and conquer — always look at the middle of a sorted array. If target is smaller, search left half; if larger, search right half. Time complexity: O(log n).

📖
Analogy: Finding a word in a sorted dictionary. You open to the middle — if your word comes before, you search the left half; after, the right half. Much faster than checking page by page!
binary_search.py
def binary_search(arr, target): left, right = 0, len(arr) - 1 while left <= right: mid = (left + right) // 2 if arr[mid] == target: return mid elif arr[mid] < target: left = mid + 1 else: right = mid - 1 return -1 sorted_nums = [1, 3, 5, 7, 9, 11, 13] print(binary_search(sorted_nums, 7)) # 3
Output
3
# Found 7 at index 3

Concept 3 · Complexity
Big-O Notation
Measure algorithm efficiency — how does performance scale with input size?

What is Big-O?

Big-O notation describes the worst-case growth rate of an algorithm's time or space usage relative to input size n. It lets us compare algorithms objectively, independent of hardware.

🚗
Analogy: Think of algorithm speed like travel. O(1) is a teleporter — instant regardless of distance. O(n) is walking — time grows with distance. O(n²) is walking but doubling back for each step. O(log n) is a shortcut that halves your remaining journey each time!

Graphs & Traversal
Model networks, paths, and connections — then navigate them with BFS & DFS.

What is a Graph?

A graph is a set of nodes (vertices) connected by edges. It can represent social networks, roads, web pages, or any relationship between entities. We store graphs as adjacency lists in Python.

🗺️
Analogy: Think of a city map. Each city is a node, roads are edges. Finding the shortest route between cities is a graph traversal problem — exactly what Google Maps solves every second!

Adjacency List in Python

graph.py
# Graph as adjacency list (dictionary) graph = { 'A': ['B', 'C'], 'B': ['A', 'D', 'E'], 'C': ['A', 'F'], 'D': ['B'], 'E': ['B', 'F'], 'F': ['C', 'E'] } # BFS — Breadth-First Search from collections import deque def bfs(graph, start): visited, queue = set(), deque([start]) visited.add(start) while queue: node = queue.popleft() print(node, end=" ") for n in graph[node]: if n not in visited: visited.add(n) queue.append(n) bfs(graph, 'A')
Output (BFS from A)
A B C D E F
# Level-by-level, closest nodes first

Apply your algorithm knowledge with these practical problems.
1
Implement Selection Sort
Write a Python function selection_sort(arr) that sorts a list by repeatedly finding the minimum element from the unsorted portion and placing it at the beginning. Print the array after each pass for the input [64, 25, 12, 22, 11]. Expected final output: [11, 12, 22, 25, 64].
💡 Hint: Find min index in unsorted portion, then swap with position i
solution_1.py
def selection_sort(arr): n = len(arr) for i in range(n): min_idx = i for j in range(i+1, n): if arr[j] < arr[min_idx]: min_idx = j arr[i], arr[min_idx] = arr[min_idx], arr[i] print(f"Pass {i+1}: {arr}") return arr selection_sort([64, 25, 12, 22, 11])

Count Operations in Linear vs Binary Search
Modify both linear_search and binary_search to count and return the number of comparisons made. Test both on a sorted list of 1000 elements searching for the last element. Compare the counts and explain why binary search wins.
💡 Hint: Add a counter variable, increment it inside each comparison, return (index, count)
solution_2.py
def linear_search_count(arr, target): count = 0 for i, v in enumerate(arr): count += 1 if v == target: return i, count return -1, count def binary_search_count(arr, target): l, r, count = 0, len(arr)-1, 0 while l <= r: count += 1 m = (l+r)//2 if arr[m] == target: return m, count elif arr[m] < target: l = m+1 else: r = m-1 return -1, count data = list(range(1, 1001)) print("Linear:", linear_search_count(data, 1000)[1], "ops") # 1000 print("Binary:", binary_search_count(data, 1000)[1], "ops") # ~10

DFS Path Finder
Implement a DFS-based function find_path(graph, start, end) that returns a list representing a path from start to end in the graph. Use the adjacency list: A–B–C–D–E–F from the examples above. Find the path from 'A' to 'F'.
💡 Hint: Use recursion or a stack. Track visited nodes and build path list
solution_3.py
graph = {'A':['B','C'],'B':['A','D','E'], 'C':['A','F'],'D':['B'],'E':['B','F'],'F':['C','E']} def find_path(graph, start, end, path=[]): path = path + [start] if start == end: return path for node in graph[start]: if node not in path: result = find_path(graph, node, end, path) if result: return result return None print(find_path(graph, 'A', 'F')) # ['A', 'C', 'F']
