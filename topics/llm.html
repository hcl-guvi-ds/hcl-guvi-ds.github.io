The big picture

From text to text, in five stages

Every response from an LLM is just this loop, running thousands of times per second.

01
Tokenize
Text is split into tokens & mapped to integer IDs.
02
Embed
Each ID becomes a high-dimensional vector encoding meaning.
03
Transformer
Dozens of attention layers mix context across tokens.
04
Predict
A softmax produces a probability for every token in the vocabulary.
05
Repeat
Sampled token is appended; the whole thing runs again.
Step 01 — Tokenization

Words become tokens

Before an LLM sees text, it breaks it into pieces called tokens. Each token is mapped to a unique number — the model never sees letters, only IDs.

Step 02 — Training

Billions of guesses, slowly less wrong

Training shows the model trillions of token sequences. Each time it predicts the next token, we measure the error (loss) and nudge billions of weights to do better. Repeat for weeks.

The whole magic: pick the next token

An LLM doesn't write sentences — it writes one token, then feeds the result back in and writes the next one. Adjust temperature to make it confident or creative.
