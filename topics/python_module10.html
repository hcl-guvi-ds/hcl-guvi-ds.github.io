Concept 1 · Classes & Objects
Classes & Objects
A class is a blueprint; an object is a real thing built from it.

What is a Class?

A class is a template that defines attributes (data) and methods (behaviour). An object is a specific instance created from that class — it has its own unique values for those attributes.

🏠
Real-life analogy: A class is like a house blueprint. The blueprint describes the number of rooms, doors, and windows — but it isn't a real house. Every house built from the blueprint is an object — each with its own address, colour, and owner.

Python Syntax

car.py
class Car: # __init__ is the constructor — called when creating an object def __init__(self, brand, speed): self.brand = brand self.speed = speed def describe(self): return f"{self.brand} goes at {self.speed} km/h" # Creating objects (instances) car1 = Car("Toyota", 120) car2 = Car("BMW", 200) print(car1.describe()) print(car2.describe())
Output
Toyota goes at 120 km/h
BMW goes at 200 km/h

Concept 2 · Encapsulation
Encapsulation
Bundle data and methods together — control what the outside world can see.

What is Encapsulation?

Encapsulation means keeping an object's internal data private and only exposing what's needed through public methods (getters/setters). This protects data from accidental modification.

🏧
Real-life analogy: An ATM machine is a perfect example. You can interact with your bank balance (view, withdraw), but you cannot directly edit the database. The internal data is hidden — you access it only through controlled methods.

Python Syntax

bank_account.py
class BankAccount: def __init__(self, owner, balance): self.owner = owner # public self._balance = balance # protected (_) self.__pin = 1234 # private (__) def deposit(self, amount): self._balance += amount def get_balance(self): return self._balance acc = BankAccount("Vinodhini", 5000) acc.deposit(1500) print(acc.get_balance()) # 6500
Output
6500

🔐 Access Modifiers — Click to Reveal

🟢
Public
No prefix
Public attributes/methods are accessible from anywhere — inside the class, subclasses, or outside.

self.name = "Alice"
obj.name # ✅ works
🟡
Protected
Single underscore _
Protected means "internal use" — accessible but by convention should not be used outside the class or subclasses.

self._balance = 5000
obj._balance # ⚠️ discouraged
🔴
Private
Double underscore __
Private uses name mangling. Truly hidden — you cannot access it directly from outside the class.

self.__pin = 1234
obj.__pin # ❌ AttributeError

Concept 3 · Inheritance
Inheritance
Reuse and extend existing classes — the "is-a" relationship in code.

What is Inheritance?

Inheritance lets a child class inherit attributes and methods from a parent class, then add or override them. This promotes code reuse and creates a logical hierarchy.

👨‍👩‍👧
Real-life analogy: A child inherits traits from their parents — eye colour, height, last name. But they also have their own unique traits and can override behaviours (e.g., they might cook differently). Child classes work the same way!

Python Syntax

animals.py
class Animal: # Parent def __init__(self, name): self.name = name def speak(self): return "Some sound" class Dog(Animal): # Child — inherits Animal def speak(self): # Override parent method return f"{self.name} says: Woof!" class Cat(Animal): # Another child class def speak(self): return f"{self.name} says: Meow!" d = Dog("Rex") c = Cat("Luna") print(d.speak(), c.speak())
Output
Rex says: Woof! Luna says: Meow!

🔗 Using super()

super() lets a child class call the parent class's method — useful to extend rather than completely replace behaviour.

super_demo.py
class Vehicle: def __init__(self, brand): self.brand = brand class ElectricCar(Vehicle): def __init__(self, brand, battery_kw): super().__init__(brand) # Call Vehicle.__init__ self.battery_kw = battery_kw ev = ElectricCar("Tesla", 100) print(ev.brand, ev.battery_kw)
Output
Tesla 100

Advanced Concept · Polymorphism
Polymorphism
One interface, many forms — the same method behaves differently for different objects.

What is Polymorphism?

Polymorphism means "many forms". In Python, the same method name can behave differently depending on the object calling it. This lets you write generic code that works with any compatible object.

🔌
Real-life analogy: A USB port is polymorphic. You can plug in a keyboard, mouse, or flash drive — each device is different, but they all connect through the same interface. Your computer says "connect()" and each device responds in its own way.
Method Overriding
Duck Typing
Abstract Classes
polymorphism.py
class Shape: def area(self): return 0 class Circle(Shape): def __init__(self, r): self.r = r def area(self): return 3.14 * self.r ** 2 class Rectangle(Shape): def __init__(self, w, h): self.w = w; self.h = h def area(self): return self.w * self.h for s in [Circle(5), Rectangle(4, 6)]: print(s.area()) # same call, different result!
Output
78.5
24
duck_typing.py
# Duck typing: "If it quacks like a duck, it IS a duck" class Dog: def sound(self): return "Woof" class Duck: def sound(self): return "Quack" def make_sound(animal): # No type check needed! print(animal.sound()) make_sound(Dog()) # Woof make_sound(Duck()) # Quack
Output
Woof
Quack
abstract.py
from abc import ABC, abstractmethod class Shape(ABC): # Abstract base class @abstractmethod def area(self): pass # Must implement in child class Triangle(Shape): def __init__(self,b,h): self.b=b; self.h=h def area(self): return 0.5*self.b*self.h t = Triangle(10,6) print(t.area()) # 30.0
Output
30.0

✏️ Exercises
Coding Challenges
Apply what you've learned with these practical OOP exercises.
1
Build a Student Class
Create a Student class with attributes name, roll_no, and marks. Add a method grade() that returns "Pass" if marks ≥ 50, else "Fail". Create two student objects and print their grades.
💡 Hint: Use if/else inside the grade() method
solution_1.py
class Student: def __init__(self, name, roll_no, marks): self.name = name self.roll_no = roll_no self.marks = marks def grade(self): return "Pass" if self.marks >= 50 else "Fail" s1 = Student("Vinodhini", 101, 85) s2 = Student("Arjun", 102, 42) print(s1.name, "-", s1.grade()) # Vinodhini - Pass print(s2.name, "-", s2.grade()) # Arjun - Fail
2
Encapsulate a Password
Create a UserAccount class with a private attribute __password. Add methods set_password(new_pw) and check_password(pw) that returns True/False. Never expose the password directly.
💡 Hint: Use double underscore __ for the private attribute
solution_2.py
class UserAccount: def __init__(self, username, password): self.username = username self.__password = password # private! def set_password(self, new_pw): self.__password = new_pw def check_password(self, pw): return self.__password == pw u = UserAccount("vinodhini", "secure123") print(u.check_password("secure123")) # True print(u.check_password("wrong")) # False
3
Inheritance: Employee → Manager
Create an Employee class with name and salary. Then create a Manager class that inherits Employee and adds a team_size attribute. Override a describe() method to include the team size.
💡 Hint: Use super().__init__() to reuse the parent constructor
solution_3.py
class Employee: def __init__(self, name, salary): self.name = name self.salary = salary def describe(self): return f"{self.name} earns ₹{self.salary}" class Manager(Employee): def __init__(self, name, salary, team_size): super().__init__(name, salary) self.team_size = team_size def describe(self): return super().describe() + f", leads {self.team_size} people" m = Manager("Vinodhini", 95000, 12) print(m.describe())
