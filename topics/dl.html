Module 1 · Foundation

What is a Neuron?

The artificial neuron is the atom of deep learning — every network, from a simple classifier to GPT-4, is built from millions of these tiny units. Understanding one neuron means understanding the core idea behind it all.

⏱ ~8 min read🎮 3 interactive demos🧩 Beginner friendly
The Biology
Inspired by the human brain
In 1943, Warren McCulloch and Walter Pitts proposed the first mathematical model of a neuron, directly inspired by biology. A biological neuron receives electrical signals through branching dendrites, processes them in the cell body (soma), and — if the combined signal is strong enough — fires an output down the axon to the next neuron.

The artificial neuron mimics this exactly: it takes multiple numerical inputs, multiplies each by a weight (importance), sums them up, adds a bias (threshold), and passes the result through an activation function (the "firing decision").
🔬 Biological Neuron
🤖 Artificial Neuron
The Math
Two simple steps: sum, then activate
Every neuron does exactly two things:
Step 1 — Weighted sum: z = w₁x₁ + w₂x₂ + w₃x₃ + b. Each input is multiplied by its weight (positive = amplify, negative = suppress, zero = ignore). The bias b shifts the decision threshold — like a baseline activation level.
Step 2 — Activation: output = f(z). An activation function decides whether and how strongly the neuron "fires". Without it, the neuron is just linear — and stacking linear functions is useless.
z = Σᵢ (wᵢ · xᵢ) + b     output = f(z)
💡 Intuition: Think of weights as "how much do I trust this input?" and bias as "how excited am I by default?" The activation function is the neuron asking "is this signal strong enough to pass on?"

Classic Example
The Perceptron — learning a logic gate
Frank Rosenblatt's Perceptron (1958) was the first trainable neural network. With weights w₁=1, w₂=1 and bias=-1.5, it perfectly implements the AND gate — outputting 1 only when both inputs are 1. This proved that a single neuron can learn a simple decision boundary.
🎯 AND Gate Perceptron
x₁x₂ANDOutput
⚠️ The Limitation

A single perceptron can only draw a straight line to separate classes. It cannot learn XOR (not linearly separable). This limitation led to the development of multi-layer networks with hidden layers.

The "AI Winter" of the 1970s was largely caused by Minsky & Papert proving the perceptron's limitations. The comeback came with backpropagation + hidden layers in the 1980s.
Python Code
Implement a neuron from scratch
import numpy as np # A single artificial neuron def neuron(inputs, weights, bias, activation='relu'): # Step 1: weighted sum z = np.dot(inputs, weights) + bias # Step 2: activation function if activation == 'relu': return np.maximum(0, z) elif activation == 'sigmoid': return 1 / (1 + np.exp(-z)) elif activation == 'tanh': return np.tanh(z) return z # linear # Try it inputs = np.array([0.5, 0.3, 0.8]) weights = np.array([1.0, 0.5, 0.8]) bias = 0.1 output = neuron(inputs, weights, bias, activation='sigmoid') print(f"Output: {output:.4f}") # Output: 0.8909

Module 2 · Foundation

Activation Functions

Without non-linearity, stacking 100 layers collapses to a single linear equation — no more powerful than one layer. Activation functions are what give deep networks their extraordinary expressive power.

⏱ ~10 min🎮 Live plotter + vanishing gradient demo
The Problem
Why linear layers alone fail
A linear function multiplied by another linear function is still linear: f(g(x)) = a·(b·x + c) + d — just one straight line. No matter how many linear layers you stack, a neural network can only learn a linear relationship between input and output. Real-world problems — image recognition, language understanding, speech — are deeply non-linear.

The solution is an activation function applied after each linear transformation. This introduces the non-linearity that allows the network to approximate any function, learn curves, boundaries, and complex patterns.
Universal Approximation Theorem: A neural network with at least one hidden layer and a non-linear activation can approximate any continuous function to arbitrary precision — given enough neurons. This is the theoretical foundation of deep learning.

The vanishing gradient problem: when a network is deep, gradients are multiplied together as they flow backwards through layers. Sigmoid's maximum gradient is 0.25 — after 5 layers, gradients shrink to 0.25⁵ ≈ 0.001. After 10 layers, nearly zero. The early layers learn nothing.

Which to use?
Practical guide to choosing activations
Hidden layers

Use ReLU as your default. Try Leaky ReLU or ELU if you see "dead neurons". For very deep nets, GELU or Swish.

Binary output

Use Sigmoid — squashes to (0,1), interpretable as probability. Never use in hidden layers.

Multi-class output

Use Softmax — normalises outputs to sum to 1, giving a probability distribution over all classes.

Module 3 · Foundation

Neural Network Architecture

A neural network is simply neurons organised into layers. The design of those layers — how many, how wide, what connections — is the architecture. Use the builder below to design and visualise your own.

⏱ ~8 min🎮 Network builder
The Structure
Three kinds of layers, one powerful machine
Neural networks are organised into layers of neurons. Data flows from left to right — this is called the forward pass. Each connection between neurons has a weight that gets learned during training.
📥
Input Layer

Receives the raw features — pixel values, word embeddings, sensor readings. No computation happens here — it's a passthrough. The number of input neurons equals the number of features in your data.

🔀
Hidden Layers

Where all the learning happens. Each hidden neuron learns to detect a different pattern or feature combination. More layers → more abstract representations. A network with 2+ hidden layers is called "deep".

📤
Output Layer

Produces the final prediction. The shape depends on the task: 1 neuron + Sigmoid for binary classification, N neurons + Softmax for N-class, 1 neuron + Linear for regression.

Parameters
Every connection is a learnable weight
The total number of parameters = Σ (neurons_in_layer × neurons_in_next_layer + bias_per_neuron). A network with 784 inputs → 256 → 128 → 10 has (784×256+256) + (256×128+128) + (128×10+10) = 235,146 parameters. Modern LLMs have billions.
import torch.nn as nn model = nn.Sequential( nn.Linear(784, 256), # 784×256 + 256 = 200,960 params nn.ReLU(), nn.Linear(256, 128), # 256×128 + 128 = 32,896 params nn.ReLU(), nn.Linear(128, 10), # 128×10 + 10 = 1,290 params nn.Softmax(dim=1) ) print(sum(p.numel() for p in model.parameters())) # 235,146

Module 4 · Core Algorithm

Backpropagation

Backpropagation is the algorithm that makes training possible. It efficiently computes how much every single weight in the network contributed to the prediction error — so they can all be corrected in one backward pass.

⏱ ~12 min🎮 Step-through animation + gradient calculator
The Big Idea
Credit assignment — who is responsible for the error?
Imagine a network makes a wrong prediction. The error is clear at the output — but which weights in which layers caused it? This is the credit assignment problem. Backpropagation solves it using the chain rule of calculus, computing ∂Loss/∂weight for every weight simultaneously.

The chain rule says: if y depends on z which depends on x, then dy/dx = (dy/dz) × (dz/dx). Backprop just applies this repeatedly from the output layer back to the input layer — hence "back-propagation of errors".
Historical note: Backpropagation was popularised by Rumelhart, Hinton & Williams in 1986. This was the key breakthrough that ended the first AI winter and launched modern deep learning.

⛰️ Loss Surface (2D)

The green dot is your current ŷ. The parabola is L=(y−ŷ)². Gradient descent moves the dot towards the minimum.

The Math
Chain rule — step by step
1. Forward Pass
ŷ = f(Wx + b)

Run data through the network. Record all intermediate values — we need them for the backward pass.

2. Compute Loss
L = (y − ŷ)²

MSE loss. The larger the error, the steeper the gradient. Loss = 0 is perfect prediction.

3. Output Gradient
∂L/∂ŷ = −2(y−ŷ)

How does loss change w.r.t. output? This is the error signal that flows backward.

4. Chain Rule
∂L/∂w = ∂L/∂ŷ · ∂ŷ/∂w

Multiply gradients at each step. For a weight w, ∂ŷ/∂w = x (the input to that neuron).

5. Weight Update
w ← w − α·∂L/∂w

Nudge every weight in the direction that reduces loss. α (learning rate) controls step size.

6. Repeat
for epoch in epochs: ...

Repeat forward + backward pass for each mini-batch until the loss converges to a minimum.

Module 5 · Core Algorithm

Training & Optimization

Training is the process of finding the best weights by minimising the loss function. The optimizer decides how you descend that loss landscape — and the choice makes a big difference in speed and final quality.

⏱ ~10 min🎮 Live gradient descent + optimizer race + overfitting demo
The Loss Landscape
Training = finding the lowest point in a high-dimensional valley
Imagine a hilly landscape where each point is a set of weights, and the height is the loss. Training is like being dropped at a random point and trying to reach the lowest valley. Gradient descent does this by always moving in the direction of the steepest downhill slope — the negative gradient.

🚀 SGD vs Momentum vs Adam — Race to the Minimum
SGD
w -= α·∇L

Pure gradient step. Simple, but slow on flat areas and oscillates in narrow valleys.

Momentum
v = β·v − α·∇L

Accumulates velocity like a ball rolling downhill. Faster through flat regions, less oscillation.

Adam
Adaptive lr per param

Combines momentum + per-parameter learning rates. Usually fastest and most robust. Default choice for most tasks.

Underfitting — model too simple, high training AND validation loss (high bias). Overfitting — model memorises training noise, low training loss but high validation loss (high variance). The goal is the sweet spot in between.

L2 Regularisation
L_total = L + λ·Σw²

Penalises large weights. Keeps the model simple and smooth.

Dropout
p=0.2 to 0.5

Randomly zeros neurons during training. Forces the network to learn redundant representations.

Early Stopping
monitor val_loss

Stop training when validation loss stops improving. Simple and very effective in practice.

Module 6 · Architecture

Convolutional Neural Networks

CNNs revolutionised computer vision. Instead of connecting every pixel to every neuron, they use small learnable filters that slide over the image — detecting edges, textures, and shapes at every location automatically.

⏱ ~12 min🎮 Draw & convolve demo
The Problem with Fully Connected Layers
Images have too many pixels for dense connections
A 224×224 RGB image has 150,528 values. A fully-connected hidden layer with 1,000 neurons would need 150 million weights — for just one layer. CNNs solve this with three key ideas: local receptive fields (each filter only looks at a small patch), weight sharing (the same filter is reused across all positions), and translation invariance (a cat is a cat whether it's top-left or bottom-right).
The convolution operation slides a small filter (kernel) over the input image. At each position, it computes the dot product between the filter and the image patch — producing one value in the feature map. Try it yourself:

What's happening: The 3×3 kernel slides over every possible position in the 8×8 grid (36 positions total). At each position, it computes a weighted sum of the 9 pixels underneath. Bright green pixels in the feature map = the kernel found its pattern there strongly.
A typical CNN stacks: Conv → ReLU → Pool → Conv → ReLU → Pool → Flatten → Dense → Softmax. Each Conv+Pool block halves spatial size and doubles depth — learning more complex features at each level.
📐 CNN Architecture Pipeline
Output Size Formula
((W − F + 2P) / S) + 1

W=input, F=filter size, P=padding, S=stride

Feature Hierarchy

Layer 1: edges. Layer 2: textures. Layer 3: shapes. Layer N: object parts. Final: complete objects.

Famous Architectures

VGG-16, ResNet-50, EfficientNet, ViT (Vision Transformer). All built on the CNN idea.

Pooling reduces spatial dimensions, making the network faster and more robust to small shifts. Max pooling keeps the strongest signal (most common). Average pooling smooths. Both use a sliding window — typically 2×2 with stride 2, halving each spatial dimension.

Module 7 · Architecture

RNNs & LSTMs

Standard neural networks process one input independently, with no memory of what came before. RNNs maintain a hidden state across time steps — enabling them to understand sequences, text, speech, and time-series data.

⏱ ~14 min🎮 Hidden state simulator + LSTM gate explorer
The Problem
Context matters — "bank" means different things in different sentences
"I went to the river bank" vs "I went to the bank to deposit money." A feedforward network processing the word "bank" in isolation can't tell which meaning is intended. It has no memory of what came before. RNNs solve this by passing a hidden state hₜ from one time step to the next — each step's output is influenced by all previous steps.
🔄 RNN Unrolled Through Time
Equation:
hₜ = tanh(Wₕ·hₜ₋₁ + Wₓ·xₜ + b)
hₜ is the new hidden state — it remembers ALL previous inputs.
⚡ Hidden State Simulator

Type a word — watch the hidden state h evolve at each character. Positive = red, negative = blue.

⚠️ Limitation: Gradients are multiplied at every step. After 20+ steps the gradient vanishes — the network forgets distant context. This is the fundamental problem LSTMs solve.
LSTMs (Long Short-Term Memory, Hochreiter & Schmidhuber 1997) add a cell state Cₜ — a separate "memory highway" that runs straight through time with only minor linear interactions. Three gates control what information enters, stays, and leaves this memory.
🧠 LSTM Cell — Click a Gate Card Below
👆 Click one of the gate cards below to learn exactly what it does inside the LSTM
🚪 Forget Gate
fₜ = σ(Wf·[hₜ₋₁,xₜ] + bf)

Decides what to erase from cell state. 0=forget, 1=keep.

📥 Input Gate
iₜ·tanh(Wc·[hₜ₋₁,xₜ]+bc)

Decides what new info to write to cell state.

📤 Output Gate
oₜ = σ(Wo·[hₜ₋₁,xₜ]+bo)

Decides what filtered view of cell state to output as hₜ.
