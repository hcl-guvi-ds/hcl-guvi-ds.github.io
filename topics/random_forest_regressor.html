🌳 Decision Trees Foundation
Understanding the building blocks of Random Forest
What Are Decision Trees?
A decision tree is a tree-shaped model that makes predictions by recursively splitting data based on features. It works for both classification and regression tasks.
💡 Real-Life Analogy
Imagine a flowchart for house buying: "Is price < $300k?" → "Is location good?" → "Has 3+ bedrooms?" Each yes/no question splits buyers into groups. Decision trees work exactly like this!

⚡ Ensemble Learning
Combining weak learners into powerful models
What is Ensemble Learning?
Ensemble learning combines multiple models to make better predictions than any individual model. It reduces overfitting and improves generalization by leveraging the diversity of multiple learners.
💡 Real-Life Analogy
Like a jury making a verdict: One person might be biased, but multiple jury members vote and reach consensus. Ensemble methods work the same way—multiple models vote to make better decisions!
🎮 Ensemble Strategy Picker
Bagging
Boosting
Stacking
Bagging (Bootstrap Aggregating)
Creates multiple subsets of the training data through random sampling WITH replacement, trains separate models on each subset, and averages predictions.
Ensemble Process Flow

🎯 Final Prediction
Code Example: Bagging
from sklearn.ensemble import BaggingRegressor from sklearn.tree import DecisionTreeRegressor # Create bagging ensemble base_tree = DecisionTreeRegressor(max_depth=5) bagging_model = BaggingRegressor( estimator=base_tree, n_estimators=10, bootstrap=True ) bagging_model.fit(X_train, y_train) predictions = bagging_model.predict(X_test)

🌲🌲🌲 Random Forest Regressor
The ultimate ensemble method for regression
What is Random Forest?
Random Forest combines bagging with random feature selection. Each tree is trained on a bootstrap sample of data using a random subset of features, creating diverse models that together make powerful predictions.
💡 Real-Life Analogy
Like a forest of independent weather forecasters: Each forecaster has different experiences and focuses on different weather patterns. When they all predict together, the forest is more accurate than any single forecaster!
🎮 Random Forest Parameters
More trees = better but slower
Deeper = complex patterns, shallow = generalization
Feature randomness creates diversity

Hyperparameter Tuning Best Practices
🎯 n_estimators (Number of Trees)
📏 max_depth (Tree Depth)
🌿 min_samples_split & min_samples_leaf
🎲 max_features (Feature Randomness)
n_estimators: 100-500 trees usually optimal. More trees = slower training but better performance.

max_depth: Controls tree complexity. 10-20 for regression, prevent overfitting with validation.

min_samples_split/leaf: Prevent overfitting by requiring minimum samples at each node.

max_features: 'sqrt' or 'log2' are standard; controls feature diversity per split.

Compare Random Forest with Decision Tree
Train both a single Decision Tree and a Random Forest on the same dataset. Compare their R² scores and analyze which performs better.
from sklearn.tree import DecisionTreeRegressor from sklearn.ensemble import RandomForestRegressor # Single Decision Tree dt_model = DecisionTreeRegressor(max_depth=15, random_state=42) dt_model.fit(X_train, y_train) dt_score = dt_model.score(X_test, y_test) # Random Forest rf_model = RandomForestRegressor( n_estimators=100, max_depth=15, random_state=42 ) rf_model.fit(X_train, y_train) rf_score = rf_model.score(X_test, y_test) print(f'Decision Tree R² Score: {dt_score:.4f}') print(f'Random Forest R² Score: {rf_score:.4f}') print(f'Improvement: {(rf_score - dt_score)*100:.2f}%')
