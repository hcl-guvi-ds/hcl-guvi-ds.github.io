Concept 1

What is a Function?

A function is a named, reusable block of code that performs a specific task. You define it once with def and call it as many times as you need.

☕

Real-Life Analogy: Coffee Machine

A coffee machine is a function. You press a button (call the function), it grinds beans, heats water, and brews coffee (executes the body) — then gives you coffee (returns output). You don't re-wire it every time you want coffee!

Concept 2

Parameters & Arguments

Parameters are the variables listed in the function definition. Arguments are the actual values you pass in when calling the function.

🃏 Click each card to explore parameter types

Tap any card to see how it works in Python
📋

Positional

Values matched by position

def add(a, b):
  return a + b

add(3, 5) # a=3, b=5
🏷️

Default Values

Optional params with fallbacks

def greet(name, msg="Hello"):
  return msg + " " + name

greet("Alice") # uses default
🔑

Keyword Args

Named values, any order

def profile(name, age, city):
  pass

profile(age=25, name="Bob", city="NYC")
📦

*args

Any number of positional args

def total(*nums):
  return sum(nums)

total(1,2,3,4,5) # → 15
📚

**kwargs

Any number of keyword args

def info(**details):
  for k,v in details.items():
    print(k, ":", v)

info(name="Alice", age=25)

parameters.py
# Default parameters
def introduce(name, role="Student", city="India"):
    print(f"Hi! I'm {name}, a {role} from {city}.")

introduce("Alice")                    # uses defaults
introduce("Bob", "Engineer")         # overrides role
introduce(name="Vinodhini", role="Data Scientist", city="Chennai")
Default parameters must always come after required parameters. def fn(required, optional="default") ✅ — not the other way around.
Concept 3

Return Values

The return statement sends a value back to wherever the function was called from. Without return, the function returns None.

🏧

Real-Life Analogy: ATM Machine

When you use an ATM, you input your PIN and amount (parameters). The machine processes it (function body) and returns your cash (return value). Without the return, the ATM just runs but gives you nothing!

# Function WITH return
def add(a, b):
    return a + b

result = add(3, 4)
print(result)      # 7
print(result * 2)  # 14 — we can use the result!

# Multiple return values (as tuple)
def min_max(numbers):
    return min(numbers), max(numbers)

lo, hi = min_max([3, 1, 9, 4, 7])
print(f"Min: {lo}, Max: {hi}")  # Min: 1, Max: 9

Concept 4

Scope & Variables

Scope determines where a variable can be accessed. Variables inside a function are local — they don't exist outside it. Variables outside are global.

🏠

Real-Life Analogy: House Rooms

Think of global variables as furniture in the living room — everyone in the house can see it. Local variables are like items in a locked bedroom — only the person in that room can use them, and they disappear when they leave.

The local scope lives only while the function runs, then vanishes!
scope.py
name = "Global Alice"    # GLOBAL variable

def show_scope():
    name = "Local Bob"   # LOCAL variable — shadows global
    print(f"Inside:  {name}")

show_scope()              # Inside:  Local Bob
print(f"Outside: {name}")  # Outside: Global Alice

# Using global keyword
counter = 0
def increment():
    global counter
    counter += 1

increment(); increment()
print(counter)  # 2
Best practice: Avoid using global variables inside functions. Instead, pass values as parameters and return results — this makes functions predictable and testable.

Lambda & Recursion

Python has lambda — anonymous one-line functions — and recursion, where a function calls itself to solve smaller sub-problems.

advanced_functions.py
# Lambda — anonymous one-line function
square = lambda x: x ** 2
print(square(5))    # 25

# Lambda with map()
nums = [1, 2, 3, 4, 5]
squares = list(map(lambda x: x**2, nums))
print(squares)       # [1, 4, 9, 16, 25]

# Recursion — function calling itself
def factorial(n):
    if n <= 1:
        return 1              # base case
    return n * factorial(n - 1)  # recursive call

print(factorial(5))   # 120
Every recursive function needs a base case — a condition where it stops calling itself. Without it, you get infinite recursion and a RecursionError!

Exercise 01Temperature Converter

Write a function celsius_to_fahrenheit(celsius) that converts Celsius to Fahrenheit using the formula: F = (C × 9/5) + 32. Test with 0, 100, and 37.

Use an f-string to format the output nicely: f"0°C = 32.0°F"
solution.py
def celsius_to_fahrenheit(celsius):
    fahrenheit = (celsius * 9/5) + 32
    return fahrenheit

print(f"0°C = {celsius_to_fahrenheit(0)}°F")    # 32.0
print(f"100°C = {celsius_to_fahrenheit(100)}°F") # 212.0
print(f"37°C = {celsius_to_fahrenheit(37)}°F")  # 98.6
Exercise 02Palindrome Checker

Write a function is_palindrome(word) that returns True if the word reads the same forwards and backwards. Test with "racecar", "hello", "level".

solution.py
def is_palindrome(word):
    word = word.lower()          # make case-insensitive
    return word == word[::-1]   # compare to reversed

print(is_palindrome("racecar"))  # True
print(is_palindrome("hello"))    # False
print(is_palindrome("level"))    # True
Exercise 03Grade Calculator with *args

Write a function average_grade(*scores) using *args that takes any number of scores and returns the average. Return the letter grade too (A≥90, B≥80, C≥70, D≥60, F otherwise).

solution.py
def average_grade(*scores):
    avg = sum(scores) / len(scores)
    if   avg >= 90: grade = "A"
    elif avg >= 80: grade = "B"
    elif avg >= 70: grade = "C"
    elif avg >= 60: grade = "D"
    else:          grade = "F"
    return avg, grade

avg, letter = average_grade(85, 92, 78, 90)
print(f"Average: {avg:.1f} → Grade: {letter}")
