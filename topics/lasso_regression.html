Lasso Regression (L1)

The scalpel of regularisation methods. Lasso doesn't just shrink coefficients — it eliminates irrelevant features entirely by setting their coefficients to exactly zero, producing a clean, interpretable sparse model.

✂️ Concept 1 of 3

Why Do We Need Lasso?

In real datasets, many features are irrelevant or redundant. OLS keeps them all and overfits. Lasso solves this by acting as an automatic feature selector — setting useless coefficients to exactly zero.

✂️

Analogy — Marie Kondo method: OLS keeps every item in the house (all features), even useless ones — and charges rent for each one. Lasso is Marie Kondo: it asks "Does this feature spark prediction joy?" If not, it goes to exactly zero. The result is a tidy, sparse model where every retained feature truly matters.

💻 Lasso Eliminates Features Code
python
import numpy as np
from sklearn.linear_model import Lasso

np.random.seed(42)
n, p = 200, 10
X = np.random.randn(n, p)

# True model: only features 0, 3, 7 matter
true_coefs = np.zeros(p)
true_coefs[[0, 3, 7]] = [5, -4, 3]
y = X @ true_coefs + np.random.randn(n)

lasso = Lasso(alpha=0.5)
lasso.fit(X, y)

for i, c in enumerate(lasso.coef_):
    status = "KEPT" if c != 0 else "ZEROED OUT"
    print(f"Feature {i:2d}: {c:+.3f}  [{status}]")

print(f"\nNon-zero features: {np.sum(lasso.coef_ != 0)}/{p}")

🪜 Why Lasso Creates Exact Zeros — Step by Step
1
OLS minimises RSS alone
OLS finds coefficients that perfectly minimise residuals. All features get non-zero coefficients, even irrelevant ones — absorbing noise. This causes overfitting.
2
Lasso adds the L1 penalty: λΣ|βj|
L1 uses absolute values, not squares. This creates a "diamond-shaped" constraint region in coefficient space. The diamond has corners on the axes.
3
The solution hits the corner of the diamond
The RSS contours (ellipses) tend to first touch the L1 constraint diamond at its corners — which are exactly on the axes. This means one or more βj = 0 exactly.
4
The subgradient creates a "dead zone" at zero
The L1 function |β| has a subgradient [-1,+1] at β=0. If the RSS gradient is smaller than λ, the coefficient is pulled to and held at exactly zero. L2 (Ridge) uses β² which has gradient 2β → 0 as β→0, so it never reaches zero.
🎯 Concept 2 of 3

Sparsity & Exact Zeros

Lasso's most powerful property: it produces sparse models — models where most coefficients are exactly zero. This enables simultaneous prediction and feature selection.

💎

Analogy — Diamond vs Sphere constraint: Ridge constraint is a sphere — smooth everywhere, no corners. The OLS ellipses can only touch a sphere tangentially (no exact zeros). Lasso constraint is a diamond — sharp corners sit exactly on axes. OLS ellipses are far more likely to first hit these corners, setting coefficients to exactly zero.

💻 Lasso for Feature Selection Pipeline Code
python
from sklearn.linear_model import Lasso, LassoCV
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
from sklearn.datasets import fetch_california_housing
import pandas as pd, numpy as np

data = fetch_california_housing()
X, y = data.data, data.target

# LassoCV finds optimal alpha automatically
lasso_pipe = Pipeline([
  ('scaler', StandardScaler()),
  ('lasso',  LassoCV(cv=5, random_state=42))
])
lasso_pipe.fit(X, y)

coefs = lasso_pipe['lasso'].coef_
selected = pd.Series(coefs, index=data.feature_names)
print("Non-zero features:")
print(selected[selected != 0].sort_values())
print(f"\nZeroed: {(coefs==0).sum()}/{len(coefs)}")
print(f"Best alpha: {lasso_pipe['lasso'].alpha_:.4f}")

🧩 Key Lasso Properties — Click to Reveal
💯
Exact Zeros
Unique to L1: Lasso can set coefficients to EXACTLY zero, unlike Ridge which only approaches zero. This is due to the non-smooth nature of |β| at β=0, creating a subgradient region that traps solutions at zero.
👆 Click to reveal
📦
Sparse Models
Automatic feature selection: In a 1000-feature dataset, Lasso might keep only 50. This sparsity makes models interpretable, faster, and less prone to overfitting. It also helps with the "curse of dimensionality".
👆 Click to reveal
🎯
Coordinate Descent
No closed-form solution: Unlike Ridge (which has b=(X'X+λI)^-1 X'y), Lasso has no analytical solution. It uses coordinate descent — optimising one coefficient at a time while holding others fixed. The soft-thresholding operator handles the non-smooth L1 penalty.
👆 Click to reveal
⚠️
Grouped Features
Lasso's weakness: When features are highly correlated, Lasso arbitrarily picks one and zeros out the rest. This can be misleading — the "chosen" feature might not be the most important. Ridge handles correlated features more gracefully by shrinking all together.

⚔️ Concept 3 of 3

Lasso vs Ridge vs ElasticNet

Both Lasso and Ridge regularise, but their penalties behave fundamentally differently. Knowing when to use each is the key skill for applied machine learning.

⚖️

Analogy — Two different diet plans: Ridge (L2) is a calorie-reduction diet — everyone eats a bit less (all coefficients shrink proportionally). Lasso (L1) is an elimination diet — some foods are completely banned (coefficients go to zero) while others are untouched. Ridge gives smaller portions to everyone; Lasso evicts the weakest guests entirely.

💡 Decision rule: Use Lasso when you believe only a few features truly matter and want an interpretable sparse model. Use Ridge when all features contribute and you mainly need to handle multicollinearity. Use ElasticNet when both are true.
⚡ Advanced Concept

LassoCV & Regularisation Paths

LassoCV automatically finds the best lambda using cross-validation. The regularisation path shows how all coefficients shrink as lambda increases — revealing the order in which features are eliminated.

🗺️

Analogy — Evacuation order: As lambda (danger level) increases, features evacuate one by one — the weakest leave first, the strongest stay longest. The regularisation path is the evacuation timeline: you can see exactly when each feature was eliminated and in what order. The last surviving features are the most important ones.

💻 LassoCV — Optimal Alpha Code
python
from sklearn.linear_model import LassoCV, lasso_path
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
import numpy as np

lasso_cv = Pipeline([
  ('sc', StandardScaler()),
  ('lasso', LassoCV(
      cv=5,
      n_alphas=100,          # try 100 alpha values
      max_iter=10000,
      random_state=42
  ))
])
lasso_cv.fit(X_tr, y_tr)

best_a = lasso_cv['lasso'].alpha_
coefs  = lasso_cv['lasso'].coef_
n_zero = np.sum(coefs == 0)

print(f"Best alpha:    {best_a:.6f}")
print(f"Train R²:      {lasso_cv.score(X_tr, y_tr):.4f}")
print(f"Test  R²:      {lasso_cv.score(X_te, y_te):.4f}")
print(f"Features kept: {len(coefs)-n_zero}/{len(coefs)}")
print(f"Sparsity:      {n_zero/len(coefs)*100:.0f}% zeroed")

⚔️ When Does Lasso Beat Ridge?
📦
Sparse True Model
Lasso wins: When only a few features truly matter (sparse ground truth), Lasso correctly identifies them by zeroing out the rest. Ridge keeps all features and dilutes signal across the noise.
👆 Click to reveal
📖
Interpretability
Lasso wins: When you need to explain your model to stakeholders — "these 5 features drive the prediction" is much cleaner than "all 100 features contribute a little." Lasso's sparsity enables this clarity.
👆 Click to reveal
🔗
Correlated Features
Ridge wins: When x1 and x2 are correlated (both truly matter), Lasso arbitrarily picks one and zeros the other. Ridge shrinks both sensibly. ElasticNet is best here — combining L1 sparsity with L2 grouping.
👆 Click to reveal
📉
High Dimensions
Lasso wins big: When p >> n (more features than samples — genomics, text NLP), Lasso is essential. Ridge keeps all features and can't handle p > n well. Lasso's sparsity is the only way to get sensible estimates.

Lasso for Sparse Feature Recovery
Create a 500-sample dataset with 50 features where only features 2, 15, and 37 truly matter (coefficients = 8, -5, 4). Add Gaussian noise. Train Lasso and check whether it correctly identifies exactly those 3 features as non-zero.
python
import numpy as np
from sklearn.linear_model import LassoCV
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
from sklearn.model_selection import train_test_split

np.random.seed(42)
n, p = 500, 50
true_idx = [2, 15, 37]
true_vals = [8, -5, 4]

X = np.random.randn(n, p)
beta = np.zeros(p)
for i, v in zip(true_idx, true_vals):
    beta[i] = v
y = X @ beta + np.random.randn(n)

X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.2)

pipe = Pipeline([
    ('sc', StandardScaler()),
    ('lasso', LassoCV(cv=5, random_state=42))
])
pipe.fit(X_tr, y_tr)

coefs = pipe['lasso'].coef_
found = np.where(coefs != 0)[0].tolist()
print(f"True features:  {true_idx}")
print(f"Lasso found:    {found}")
print(f"Match: {set(found) == set(true_idx)}")
print(f"Test R²: {pipe.score(X_te, y_te):.4f}")

Compare Lasso vs Ridge on Correlated Features
Create a dataset where x1 and x2 are 95% correlated and both matter equally. Show that Lasso arbitrarily picks one while Ridge shrinks both. Report coefficients and test R² for both. Explain which is more appropriate here.
python
import numpy as np
from sklearn.linear_model import Ridge, Lasso
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import train_test_split

np.random.seed(42)
n = 300
x1 = np.random.randn(n)
x2 = 0.95*x1 + 0.05*np.random.randn(n)  # 95% correlated
x3 = np.random.randn(n)
# Both x1 AND x2 equally important!
y  = 3*x1 + 3*x2 + 5*x3 + np.random.randn(n)
X  = np.column_stack([x1, x2, x3])

X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.2)
sc = StandardScaler()
Xtr_s = sc.fit_transform(X_tr)
Xte_s = sc.transform(X_te)

print("True: x1=3, x2=3, x3=5 (x1,x2 both matter equally)")
for name, m in [('Ridge', Ridge(1.0)),('Lasso', Lasso(0.1))]:
    m.fit(Xtr_s, y_tr)
    print(f"{name}: coefs={m.coef_.round(2)}  R²={m.score(Xte_s,y_te):.3f}")
# Ridge keeps both x1,x2 similarly shrunk
# Lasso zeroes one of them — misleadingly!

Regularisation Path — Plot Coefficient Traces
Use sklearn's lasso_path() to compute coefficients at many alpha values. Print a text-based visualisation showing which features survive at alpha = 0.001, 0.01, 0.1, 1.0, and 10.0. Identify the "last survivor" (most important feature).
python
from sklearn.linear_model import lasso_path
from sklearn.preprocessing import StandardScaler
from sklearn.datasets import fetch_california_housing
import numpy as np

data = fetch_california_housing()
X_s = StandardScaler().fit_transform(data.data)

alphas = [0.001, 0.01, 0.1, 1.0, 10.0]
alpha_full, coefs_full, _ = lasso_path(X_s, data.target,
                                       alphas=alphas)
print(f"{'Feature':12s}", end="")
for a in alphas:
    print(f"  a={a:.3f}", end="")
print()
for i, feat in enumerate(data.feature_names):
    print(f"{feat:12s}", end="")
    for j in range(len(alphas)):
        c = coefs_full[i, j]
        sym = f"{c:+.3f}" if abs(c) > 1e-4 else "  ZERO"
        print(f"  {sym:7s}", end="")
    print()

last_alive = [(feat, np.max(np.abs(coefs_full[i])))
              for i,feat in enumerate(data.feature_names)]
last_alive.sort(key=lambda x: -x[1])
print(f"\nMost important: {last_alive[0][0]}")
