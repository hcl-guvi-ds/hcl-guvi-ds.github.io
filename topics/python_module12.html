Project 1 · Calculator App
Build a Python Calculator
Create a fully functional calculator using functions, error handling, and a menu-driven loop.

What We're Building

A command-line calculator that handles +, −, ×, ÷ operations with input validation and an infinite loop until the user chooses to quit.

🏗️
Think of it like building a vending machine: The machine (main loop) keeps running. Each button (function) does one specific job. If you press an invalid button (bad input), the machine says "Invalid choice" rather than crashing. That's how we design our calculator!

Complete Calculator Code

calculator.py
def add(a, b): return a + b def subtract(a, b): return a - b def multiply(a, b): return a * b def divide(a, b): if b == 0: raise ValueError("Cannot divide by zero!") return a / b def main(): operations = {'1': ('Add', add), '2': ('Subtract', subtract), '3': ('Multiply', multiply), '4': ('Divide', divide)} while True: print("\n=== Calculator ===") for k, (name, _) in operations.items(): print(f" {k}. {name}") print(" 5. Exit") choice = input("Choose operation: ") if choice == '5': break if choice not in operations: print("Invalid choice!"); continue try: a = float(input("Enter first number: ")) b = float(input("Enter second number: ")) _, func = operations[choice] print(f"Result: {func(a, b)}") except (ValueError, TypeError) as e: print(f"Error: {e}") main()

Complete To-Do App Code

todo_app.py
import json def load_tasks(filename="tasks.json"): try: with open(filename) as f: return json.load(f) except FileNotFoundError: return [] def save_tasks(tasks, filename="tasks.json"): with open(filename, 'w') as f: json.dump(tasks, f, indent=2) def add_task(tasks, name): tasks.append({"name": name, "done": False}) def view_tasks(tasks): if not tasks: print("No tasks yet!"); return for i, t in enumerate(tasks, 1): status = "✅" if t["done"] else "⬜" print(f" {i}. {status} {t['name']}") def complete_task(tasks, idx): tasks[idx - 1]["done"] = True def delete_task(tasks, idx): tasks.pop(idx - 1) # Main loop — try it yourself! tasks = load_tasks() add_task(tasks, "Learn Python Projects") add_task(tasks, "Build Calculator") complete_task(tasks, 2) view_tasks(tasks) save_tasks(tasks)

Project 3 · Quiz Game
Build a Python Quiz Game
Use dictionaries, loops, and score tracking to create an engaging command-line quiz engine.

What We're Building

A quiz game that loads questions from a dictionary, presents multiple-choice options, tracks the score, and shows a final result with percentage.

🎮
Think of it like a game show host: The host (main loop) reads each question card (dictionary entry), shows options, waits for your answer, checks if it's right, and keeps score. At the end, reveals your result. Our Python code IS the game show host!

Complete Quiz Game Code

quiz_game.py
questions = [ {"q": "What keyword creates a function in Python?", "options": ["func", "def", "function", "lambda"], "answer": "def"}, {"q": "Which data type is mutable?", "options": ["tuple", "string", "list", "int"], "answer": "list"}, ] def run_quiz(questions): score = 0 for i, q in enumerate(questions, 1): print(f"\nQ{i}: {q['q']}") for j, opt in enumerate(q['options'], 1): print(f" {j}. {opt}") ans = input("Your answer: ").strip() if ans == q["answer"]: print("✅ Correct!"); score += 1 else: print(f"❌ Wrong! Answer: {q['answer']}") pct = score / len(questions) * 100 print(f"\n🎯 Score: {score}/{len(questions)} ({pct:.0f}%)") run_quiz(questions)

Advanced Project · File Manager
Build a Python File Manager
Use the os module, file I/O, and OOP to create a powerful command-line file management tool.

What We're Building

A class-based file manager that can list directory contents, create/delete files, search by pattern, and display file metadata — all from the command line.

🗃️
Think of it like macOS Finder or Windows Explorer: Those GUI apps interact with your file system. We're building the same functionality but in Python code, using the os module which acts as Python's bridge to your operating system.

⚡ Basic vs Class-Based Approach

See how wrapping code in a class makes it more organised and reusable.

❌ Without OOPBasic
import os

path = "."
files = os.listdir(path)
for f in files:
  print(f)

# Hard to reuse, messy
# globals everywhere
✅ With OOP (Class)Better
class FileManager:
  def __init__(self, path):     self.path = path   def list_files(self):     return os.listdir(self.path)
# Clean, reusable, scalable

Complete File Manager Code

file_manager.py
import os import glob class FileManager: def __init__(self, base_path="."): self.path = os.path.abspath(base_path) def list_files(self): items = os.listdir(self.path) for item in sorted(items): full = os.path.join(self.path, item) kind = "📁" if os.path.isdir(full) else "📄" size = os.path.getsize(full) print(f" {kind} {item} ({size} bytes)") def create_file(self, name, content=""): filepath = os.path.join(self.path, name) with open(filepath, 'w') as f: f.write(content) print(f"✅ Created: {name}") def search_files(self, pattern): matches = glob.glob(os.path.join(self.path, f"*{pattern}*")) return [os.path.basename(m) for m in matches] def delete_file(self, name): filepath = os.path.join(self.path, name) if os.path.exists(filepath): os.remove(filepath) print(f"🗑️ Deleted: {name}") # Usage fm = FileManager("/home/vinodhini") fm.list_files() fm.create_file("notes.txt", "Hello Python!") print(fm.search_files(".txt"))

Add Priority to To-Do Tasks

PRIORITY_ORDER = {"High": 0, "Medium": 1, "Low": 2} PRIORITY_ICONS = {"High": "🔴", "Medium": "🟡", "Low": "🟢"} def add_task(tasks, name, priority="Medium"): tasks.append({ "name": name, "done": False, "priority": priority }) def view_tasks(tasks): sorted_tasks = sorted( tasks, key=lambda t: PRIORITY_ORDER.get(t["priority"], 1) ) for i, t in enumerate(sorted_tasks, 1): done = "✅" if t["done"] else "⬜" icon = PRIORITY_ICONS[t["priority"]] print(f" {i}. {done} {icon} {t['name']}")
